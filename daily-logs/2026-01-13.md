## üß† What I Learned / Key Takeaways
- `dpkg` is the low-level package manager responsible for installing `.deb` packages and maintaining the authoritative database of installed software
- `apt` acts as a higher-level dependency resolver and planner, while `dpkg` performs the actual installation and logging
- `dpkg -l` output is alphabetical and cannot be used to determine install order or recency
- Accurate install timelines must be reconstructed using `/var/log/dpkg.log*` and `/var/log/apt/history.log*`
- Log rotation explains why package history appears to ‚Äústop‚Äù at a certain date; older logs are compressed (`.gz`) and must be accessed with `zless` or `zgrep`
- Installing security tools (auditd, rkhunter, AppArmor utilities) may prompt for local mail notification preferences without exposing SMTP or internet-facing services
- KVM being loaded in the kernel confirms hardware virtualization is active, but libvirt is required to manage virtual machines in a practical and persistent way
- Absence of `kvm` and `libvirt` user groups indicates libvirt is not yet installed, not a system issue
- Maintaining a default-deny inbound firewall posture (UFW) significantly reduces risk even when services are installed
