## üß† What I Learned / Key Takeaways
- Linux does not use a single ‚ÄúProgram Files‚Äù directory; software is split across `/usr/bin`, `/etc`, `/usr/lib`, `/usr/share`, etc. based on purpose  
- `$PATH` is an ordered list of directories Linux searches to find executables when a command is run  
- LibreOffice is considered a *binary* because the executable is compiled machine code (ELF), even though it loads many supporting files  
- Snap packages are containerized and sandboxed, while APT installs software natively into the system  
- Installing development tools (like VS Code) via APT is preferred for performance, predictability, and professional workflows
