## üß† What I Learned / Key Takeaways
- ***AWS Labs GitHub*** - AWS hosts hands-on tutorials and sample code at https://github.com/aws-samples/, separate from main documentation
- Easy to think of **IaC (Infrastructure as Code) as a *blueprint* of your infrastructure**
    - CloudFormation allows you to write IaC as either JSON or YAML
    - **Stack** = actual infrastructure built from template (blueprint)
    - **Template** = the blueprint file defining resources
- **AWS CloudShell storage is NOT EFS** - it's AWS-managed persistent storage (1GB per region in $HOME)
- **CloudShell** = browser-based CLI, pre-authenticated, free, perfect for quick tasks
- **ARNs (Amazon Resource Names)** uniquely identify resources: `arn:aws:service:region:account-id:resource`
- **Shared Responsibility Model** - "Security **IN** the cloud" (customer) vs "Security **OF** the cloud" (AWS)
    - Simple rule: "Can you configure it?" ‚Üí Your responsibility
    - Responsibility shifts with service type: IaaS (most) > PaaS > SaaS > FaaS (least)
- **Access Keys** = programmatic credentials (NEVER commit to code, max 2 per user)
- **AWS CLI v2** is current; v1 enters maintenance mode July 2026, end-of-support July 2027
- **Compute options strategy:** EC2 (most control) ‚Üí ECS/EKS ‚Üí Fargate ‚Üí Lambda (least management)
- **Edge Computing** = push workloads close to users (Wavelength for 5G, Local Zones for cities)
- **Hybrid Computing** = workloads on-premise AND in AWS cloud
- **HPC** = supercomputer-level computing using AWS ParallelCluster for scientific workloads
