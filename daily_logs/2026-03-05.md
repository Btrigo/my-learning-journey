## 🧠 What I Learned / Key Takeaways
- AWS security follows the **Shared Responsibility Model**, dividing security duties between AWS (security *of* the cloud) and the customer (security *in* the cloud)
- **IAM** controls identity and access to AWS resources using users, roles, groups, and policies
- **WAF** protects applications from web exploits such as SQL injection and cross-site scripting
- **AWS Shield** provides automatic protection against common DDoS attacks
- **KMS** manages encryption keys for data encryption at rest
- **ACM** manages TLS certificates used to secure HTTPS connections
- **GuardDuty** detects suspicious or malicious activity across AWS accounts
- **Inspector** performs automated vulnerability scanning on infrastructure
- **Detective** helps investigate security incidents and understand attack timelines
- **Security Hub** aggregates security findings across AWS services
- **Macie** uses machine learning to detect sensitive data stored in Amazon S3
