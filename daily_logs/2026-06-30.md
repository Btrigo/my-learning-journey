## 🧠 What I Learned / Key Takeaways
- ADUC = Active Directory Users and Computers
- RDP always needs two separate inputs: target address (Computer field) and domain-prefixed username (`DOMAIN\user`) — true across RDP, SSH, VNC
- Delegation of Control = granting limited AD privileges (e.g., password resets for one OU) without full Domain Admin — least privilege in practice
- NLA blocks forced password-change prompts over RDP since auth happens before the interactive session starts
- OU placement isn't just organizational — it directly determines what GPOs an object inherits (why Workstations/Servers/DCs get separate OUs)
- GPOs only take effect once linked to a scope; linking to root domain cascades down, but Computer Configs are ignored by user-only OUs
- SYSVOL (GPO distribution) vs NTDS.dit (AD object/credential database) — different jobs
- Kerberos: authenticate once via KDC → TGT → service tickets per resource, no repeated password transmission
- Reconnected Security+ Kerberos theory to actual hands-on use for the first time
