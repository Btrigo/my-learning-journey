## 🧠 What I Learned / Key Takeaways

- **Global services** (memorize short list): IAM, Route 53, CloudFront, WAF, Shield, AWS Organizations — everything else defaults to Regional
- **Zonal services**: EC2, EBS, Subnets
- **EBS vs EFS**: EBS = one instance, one AZ, billed on provisioned capacity; EFS = multiple instances, regional, billed on actual use, Linux only
- **Auto-scaling by default**: Lambda, DynamoDB (on-demand), S3, SQS, SNS, Fargate, Aurora Serverless, API Gateway, Cognito, EventBridge
- **Fully managed ≠ auto-scaling**: DocumentDB, Keyspaces, Neptune are fully managed but NOT auto-scaling by default
- **Security groups = stateful** (implicit deny, allow rules only); **NACLs = stateless** (explicit deny/allow, evaluated in order)
- **Support Plans — critical distinctions**:
  - Full Trusted Advisor = Business and above
  - Phone/chat support = Business and above
  - Support API = Business and above
  - Pool of TAMs = Enterprise On-Ramp
  - Designated TAM = Enterprise ONLY
  - IEM included = On-Ramp and Enterprise (extra fee at Business)
  - Trusted Advisor Priority = Enterprise ONLY
  - Business-critical response: On-Ramp = 30 min, Enterprise = 15 min
- **IEM (Infrastructure Event Management)**: Dedicated AWS support before, during, and after a major planned event — "before, during, and after" is the exam trigger phrase
- **SNS vs SQS vs EventBridge**: SNS = push/broadcast; SQS = pull/queue; EventBridge = smart routing with rules
- **Bedrock vs SageMaker**: Bedrock = consume AI models to build apps; SageMaker = build/train your own models
- **CDK vs SDK**: CDK = define infrastructure as code; SDK = your app talking to AWS at runtime
- **Answer-changing discipline reinforced**: "Designated TAM" → Enterprise, not On-Ramp (missed Q10 and Q12 by misreading keyword signals)
