## Key Takeaways
- Access key decision starts with ONE question: where is the workload?
  On-premises = keys, On AWS = role, Mobile = Cognito
- Root access keys should never exist — always wrong answer on exam
- Hardcoded keys on EC2 = always wrong — attach IAM role instead
- Key rotation = customer responsibility, every 90 days, Trusted Advisor flags it
- Gateway Endpoint = S3 and DynamoDB only, free, updates route table
- Interface Endpoint = all other services, powered by PrivateLink, has cost
- PrivateLink ≠ on-premises connectivity — it keeps traffic inside AWS network
- Direct Connect = private dedicated line, no internet, weeks to set up
- Site-to-Site VPN = encrypted tunnel over internet, hours to set up
- Aurora does NOT support Oracle or SQL Server — RDS does
- RDS classified as regional for exam purposes despite instances being zonal
- EC2 and EBS are the primary zonal examples — EBS volume must be in
  same AZ as its EC2 instance
- OpenSearch = formerly Elasticsearch, log analytics, full-text search
- Snow Family trigger = no internet or petabyte+ scale migration
