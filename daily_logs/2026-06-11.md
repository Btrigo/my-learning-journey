## 🧠 What I Learned / Key Takeaways
- GNS3 requires real Cisco IOS images to function — without them devices can't boot or be configured; Packet Tracer is self-contained and covers everything needed at this stage
- DHCP options are numbered codes that deliver extra configuration alongside an IP lease (e.g. option 3 = default gateway, option 6 = DNS, option 150 = TFTP server for Cisco IP phones); vendor documentation specifies which options each device needs
- Switches learn MAC addresses from the source MAC of any arriving frame regardless of protocol — ARP is not required; the MAC address table is purely reactive and only updates when traffic is sent
- Encapsulation = data traveling down the stack (Layer 7 → Layer 1), adding headers at each layer; decapsulation = data traveling up the stack on the receiving end, stripping headers
- ARP is IPv4-specific — it resolves IP addresses to MAC addresses at Layer 2; IPv6 replaces ARP entirely with Neighbor Discovery Protocol using ICMPv6 multicast
- APIPA (169.254.x.x /16) is automatically assigned when no DHCP server is reachable; available from Windows 98 onward; link-local only, cannot be routed across a router
- A broadcast domain is the set of all devices that receive a broadcast frame; routers separate broadcast domains, switches do not
- Port security defaults: one MAC address allowed, violation mode is shutdown; the first MAC address that sends traffic into a port is learned as the allowed address; a second MAC address triggers an immediate violation and error-disables the port
- Static MAC address configuration and sticky MAC are alternatives to dynamic learning in port security — more explicit and more secure

## 🛠 Commands / Tools Practiced
- Cisco IOS DHCP server config: `ip dhcp pool`, `network`, `default-router`, `dns-server`, `ip dhcp excluded-address`, `lease`
- Cisco IOS DHCP client config: `ip address dhcp` on an interface
- `show ip dhcp binding`, `show ip dhcp pool`
- `show mac address table dynamic`
- `show port-security`, `show port-security interface`, `show port-security address`
- `switchport mode access`, `switchport port-security`
- `shutdown` / `no shutdown` to recover an error-disabled port
- Windows: `ipconfig /release`, `ipconfig /renew`, `ipconfig /all`, `tracert`
- Linux: `ip address`, `ip route`, `tracepath`, `traceroute`
