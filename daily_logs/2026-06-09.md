## 🧠 What I Learned / Key Takeaways
- Reinforced the full DHCP configuration workflow on a Cisco router from memory — client side and server side
- Exclusion range should be configured before creating the pool to prevent unwanted address allocation on client connect
- Router's own IP must be excluded from the pool or it risks being leased to a client
- DHCP server and DHCP client can coexist on the same router on different interfaces

## 🛠 Commands / Tools Practiced
- `ip address dhcp` — configure interface as DHCP client
- `ip dhcp excluded-address <start> <end>` — exclude address range from pool
- `ip dhcp pool <name>` — create/enter DHCP pool config
- `network <network> <mask>` — define the pool's address range
- `default-router <ip>` — set the gateway pushed to clients
- `dns-server <ip>` — set DNS pushed to clients
- `lease <days>` — set lease duration
- `show ip dhcp binding` — verify IP-to-MAC lease assignments
- `show ip dhcp pool` — verify pool stats
- `show ip interface brief` — confirm interface IPs and status
- `show run interface <int>` — verify interface config
