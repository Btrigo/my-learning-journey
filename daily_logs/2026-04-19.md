### CAF Phases
- **Launch** phase = demonstrating incremental business value and gathering insights for future scaling — NOT Envision
- Envision = making the case before anything is built. Launch = delivering pilots and proving value incrementally.
- Phases in order: Envision → Align → Launch → Scale

### CAF Perspectives
- **Platform** = CTO-level, infrastructure migration, technical architecture — "building the highway"
- **Business** = ROI, financial justification, "is it worth it?"
- Trigger: CTO + migration + technical → Platform. CEO + financial value → Business.

### AWS Global Accelerator
- Routes traffic through AWS's private global network, reducing latency rather than routing over the public internet
- Provides **2 static anycast IP addresses** as a fixed entry point regardless of how many regions/servers are behind it
- Traffic enters AWS network at the nearest edge location — not at the region
- Primary exam benefit: **decreased latency**. Security (less public internet exposure) is a side effect, not the headline.
- Do NOT confuse with CloudFront: Global Accelerator = private network routing. CloudFront = content caching at edge.

### AWS Service Catalog
- Lets IT admins create a pre-approved menu (portfolio) of AWS resources that end users can self-provision
- Users can only deploy what's in the catalog — no going rogue
- Exam trigger: "limit access to predefined AWS resources", "standardized provisioning", "portfolio of approved services"

### Amazon ElastiCache
- In-memory caching service — stores frequently accessed data in RAM for microsecond retrieval
- Reduces database load and latency
- Two engines only: **Redis** (powerful, persistent, Multi-AZ) and **Memcached** (simple, fast, lightweight)
- Memory anchor: ElastiCache = RAM-based speed boost for your database. Redis = powerful. Memcached = simple.
- Apache Kafka (Amazon MSK) is a streaming service — NOT an ElastiCache engine

### AWS Fargate vs Amazon ECS
- **ECS** = container orchestration — manages where/how containers run, but you still manage underlying EC2 infrastructure
- **Fargate** = serverless compute engine for containers — no servers to manage at all
- Exam trigger: "eliminate need to manage containers manually" → Fargate, not ECS

### Amazon Redshift
- Managed **data warehouse** service — OLAP, not OLTP
- Built for complex analytical queries across massive historical datasets
- **Redshift Spectrum** = extension that queries S3 data directly without loading it into Redshift
- Memory anchor: OLTP = live transactions → RDS/Aurora/DynamoDB. OLAP = analytics/warehouse → Redshift.

### AWS IAM Identity Center vs AWS Organizations vs SCPs
- **Organizations** = manages account structure, consolidated billing, the "house"
- **SCPs** = rules that control what services can be used across accounts — the "rules of the house"
- **IAM Identity Center** = SSO — single login to access multiple AWS accounts
- Triggers: "single login" / "SSO" → IAM Identity Center. "Control services across accounts" → SCPs. "Manage/group accounts" → Organizations.

### IAM Entities
- **IAM User** = real identity with long-term credentials (password + access keys). For humans or apps needing persistent access.
- **IAM Role** = temporary identity — no long-term credentials. For AWS services, cross-account access, or temporary access.
- **IAM Group** = groups users for easier permission management. No credentials.
- **IAM Policy** = document defining permissions. No credentials.
- Memory anchor: IAM User = permanent identity with keys. IAM Role = temporary costume, no permanent credentials.

### AWS KMS vs AWS CloudHSM
- **KMS** = AWS manages everything including the hardware and the keys. Fully hands-off.
- **CloudHSM** = AWS hosts the physical hardware, but YOU manage the keys and cryptographic operations entirely. AWS cannot access your keys.
- Exam signal: "without managing your own hardware module" → KMS. "Dedicated hardware", "you control the keys" → CloudHSM.

### AWS CodeSuite
- **CodeCommit** = source code repository (like GitHub on AWS)
- **CodeBuild** = compiles source code, runs tests, produces deployment packages ← the builder
- **CodeDeploy** = deploys packages to EC2, Lambda, or on-premises
- **CodePipeline** = orchestrates the entire CI/CD pipeline end-to-end ← the conductor
- **CodeArtifact** = stores and manages software packages/dependencies
- Exam trigger: "compiles code, runs tests, produces packages" → CodeBuild. Never CodePipeline.

### AWS Step Functions
- Orchestrates multiple AWS services into visual workflows (state machines)
- Coordinates microservices and distributed applications
- Exam trigger: "coordinate components", "visual workflows", "microservices orchestration" → Step Functions

### Amazon Kinesis
- Platform for collecting, processing, and analyzing real-time streaming data at scale
- **Kinesis Data Streams** = ingest and process real-time data streams
- **Kinesis Data Firehose** = deliver streaming data to destinations like S3 or Redshift
- **Kinesis Data Analytics** = query streaming data with SQL in real-time
- AWS Glue = batch ETL/data preparation — NOT real-time streaming
- Exam trigger: "real-time streaming data", "continuous data ingestion" → Kinesis

### Origin Access Control (OAC)
- CloudFront feature that ensures S3 content can ONLY be accessed through CloudFront — not directly via S3 URL
- Exam trigger: "restrict access to S3 content via CloudFront" → OAC

### AWS Trust & Safety
- The team that handles abuse of AWS infrastructure: phishing, malware, spam, DoS/DDoS originating from AWS resources
- NOT the Concierge team (billing/account only) and NOT Architecture Support

### AWS Support Plans — Key Details
- **Support API access** = Business tier and above. Developer does NOT have it.
- **Concierge Support Team** = Enterprise only (not Enterprise On-Ramp)
- Enterprise On-Ramp gets a pool of billing specialists, NOT the full Concierge team
- "Most affordable with X" questions → find the lowest tier where X first appears

### AWS Cost Explorer
- Stores and displays **12 months** of historical cost data. Not 3, not 6, not 15. Always 12.
- Available before creating an AWS account (public version at calculator.aws)
- In-console version requires account and supports RI/Savings Plan modeling and historical usage import

### EBS Billing
- EBS charges apply in both **running AND stopped** states
- Charges stop only at **termination**
- The question "billing incurred only in a specific state" → answer is Stopped (EBS bill without EC2 bill)

### S3 Storage Classes — Retrieval Fees
- **No retrieval fees**: S3 Standard and S3 Intelligent Tiering ONLY
- **Has retrieval fees**: Standard-IA, One Zone-IA, Glacier, Glacier Deep Archive
- IA = Infrequent Access = cheaper storage but charges per retrieval

### Zonal vs Regional vs Global Services
- **Zonal**: EC2, EBS (tied to a specific AZ)
- **Regional**: S3, DynamoDB, RDS, SQS, SNS
- **Global**: IAM, Route 53, CloudFront
