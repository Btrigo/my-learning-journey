## 💡 What I Learned / Key Takeaways
- Workgroup vs. domain: domains centralize account/policy management via a Domain Controller (DC)
- Active Directory Domain Services (AD DS) organizes network objects (users, computers, groups) with attributes
- Organizational Units (OUs) group objects and scope Group Policy application
- User/computer account management via Active Directory Users and Computers (ADUC)
- Security groups vs. distribution groups; group scopes (Domain Local, Global, Universal)
- Group Policy Objects (GPOs) push centralized config to users/computers, linked at OU/domain/site level
- Kerberos is the default AD auth protocol (ticket-based via KDC); NTLM is the legacy fallback
- Domain trees, forests, and trust relationships (one-way/two-way, transitive/non-transitive)
