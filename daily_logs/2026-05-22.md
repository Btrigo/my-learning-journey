## 🧠 What I Learned / Key Takeaways
- TCP/IP 5-layer model is what's used in the real world: Physical, Data Link, Network, Transport, Application — OSI's 7 layers still referenced in conversation (e.g. "Layer 7 problem")
- Layer 1 = bits (electrical, optical, radio signals); Layer 2 = frames + MAC addresses (per-hop); Layer 3 = packets + IP addresses (end-to-end); Layer 4 = segments (TCP/UDP); Layer 7 = application protocols (HTTP, FTP, SSH, etc.)
- MAC addresses are physical/hardware-level and link-scoped — IP addresses are logical and end-to-end; Layer 2 encapsulation changes at every hop, Layer 3 does not
- TCP = connection-oriented, acknowledges delivery (3-way handshake); UDP = fire and forget, no confirmation
- Standardization (RFCs) is why multi-vendor networking works today — same reason we have USB-C across devices
