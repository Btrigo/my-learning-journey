## Key Takeaways
- Aurora is AWS's fully managed, cloud-native relational database — MySQL/PostgreSQL compatible only; never Oracle or SQL Server
- Standard Aurora is NOT serverless — Aurora Serverless is a separate configuration
- RDS is managed; Aurora is fully managed — key distinction for exam questions about operational overhead
- Serverless/auto-scale by default: Lambda, DynamoDB (on-demand), S3, SQS, SNS, EventBridge, Fargate, Aurora Serverless, API Gateway, Cognito
- Data pipeline flow: Kinesis → Firehose → S3 → Glue → Athena/Redshift
- Firehose delivers data, it does not clean or structure it — that's Glue
- OLTP (transactions, row-by-row) → RDS/Aurora; OLAP (analytics, historical) → Redshift
- CAF = organizational readiness for cloud migration; Well-Architected Framework = technical evaluation of architecture
- Well-Architected pillars (OSRPCS): Operational Excellence, Security, Reliability, Performance Efficiency, Cost Optimization, Sustainability
- X-Ray = distributed tracing across serverless/microservices architectures
- AMI = software blueprint (OS, apps); Instance type = hardware specs (CPU, memory)
