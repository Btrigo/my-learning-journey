## 🧠 What I Learned / Key Takeaways
- YAML indentation defines structure the same way JSON's `{}` braces do — sibling items in a list must share identical indentation, or the parser breaks the relationship between them
- `id-token: write` is the specific workflow permission required to request an OIDC token at all; without it, the OIDC handshake can't begin
- IAM trust policy condition operators (`StringEquals`, `StringLike`) are case-sensitive — a single-character casing mismatch between the policy and the actual GitHub account name caused a full authentication failure with a generic, unhelpful error message
- `Not authorized to perform sts:AssumeRoleWithWebIdentity` gives no indication of *which* condition failed — diagnosing it requires manually re-checking each condition value against the actual token claims
- Git Bash can mangle certain arguments (like `/*`) before passing them to the AWS CLI on Windows; PowerShell is the safer choice for these commands
- Documenting one's own code/IAM policies in a personal project write-up isn't a copyright concern — only account IDs/resource IDs were redacted, for operational security, not authorship reasons
