## 🧠 What I Learned / Key Takeaways
- Secrets Manager uses KMS to encrypt the secrets it stores
    - Secrets Manager stores your password, KMS encrypts it
- ***ACM certificates for CloudFront must be created in us-east-1 regardless of where your resources are***
- KMS keys never leave KMS in plaintext — encryption/decryption happens internally
- Envelope encryption = DEK encrypts your data, KMS key encrypts the DEK
    - Plaintext DEK is discarded immediately after use — never persisted
- ACM Public Certificates cannot export the private key — it stays locked in AWS
- ACM Private Certificates (via AWS Private CA) can export — used for non-AWS/internal resources
- Symmetric encryption = one key, fast, used for bulk data (AES-256)
- Asymmetric encryption = key pair, slower, used for key exchange and signatures (RSA, Ed25519)
- Hashing is NOT encryption — it is one-way and irreversible (SHA-256, bcrypt)
- Salting defeats rainbow table attacks — unique salt per password makes precomputed tables useless
- Digital signatures = private key signs, public key verifies — proves authenticity and integrity
- My GitHub SSH private key lives on my machine (~/.ssh/id_ed25519), public key lives on GitHub
    - Private key never travels — only the signature does
- Blind SQLi = no visible output, attacker infers data via boolean responses or time delays (SLEEP)
- XML = self-describing structured data using custom tags — JSON's older, more verbose cousin
- XXE exploits XML parser's external entity feature to read local server files
- AWS WAF blocks SQLi and XSS at the edge (CloudFront, ALB, API Gateway)
- GuardDuty = threat detection (analyzes VPC Flow Logs, DNS, CloudTrail)
- Inspector = vulnerability scanning (EC2, containers, Lambda)
- Detective = security investigation and root cause analysis
- Security Hub = centralized dashboard aggregating findings from GuardDuty, Inspector, Macie, etc.
- Amazon Macie = ML-powered sensitive data detection in S3 (PII, financial records)
- Shared Responsibility Model — AWS secures the cloud, customer secures what's in the cloud
- SageMaker = build, train, deploy custom ML models (full managed ML platform)
- Bedrock = access foundation models via single API for generative AI apps (includes Anthropic Claude)
- Amazon Q = AI assistant — Q Business for employees, Q Developer for code/debugging
- Data lake (S3) = raw unstructured data / Data warehouse (Redshift) = structured analytics data
- Kinesis = real-time streaming data ingestion and processing
- Athena = serverless SQL queries directly on S3 — no infrastructure, pay per query
- QuickSight = BI dashboards and visualizations on top of analytics data
- ETL = Extract (collect) → Transform (clean/reshape) → Load (store)
- Lambda in data pipelines = transformation worker (e.g., JSON → CSV for downstream compatibility)
- "Key" is an overloaded term across cryptography, programming, and databases — context always matters
