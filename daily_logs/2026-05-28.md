## 🧠 What I Learned / Key Takeaways
- Hubs and repeaters = Layer 1 — dumb devices, flood traffic out all ports except the incoming port, no MAC address table. A hub is a multiport repeater
- Bridges and switches = Layer 2 — use MAC address tables to intelligently forward frames to the correct port only
- Pure L2 switch (e.g. 2950) only uses an IP address for management (SSH access), cannot route between VLANs or networks
- L3/multilayer switch can run routing protocols (OSPF, EIGRP, RIP), route between VLANs, and create ACLs — essentially a combined switch and router
- Managed vs unmanaged switches — unmanaged = plug in and works, no config possible, no console port. Managed = fully configurable via console
- Routers = Layer 3 — use IP addresses to route between networks. Modern routers often have built-in switch ports, blurring the device lines
- WAN connections are mostly Ethernet now — routing Ethernet to Ethernet, or Ethernet to 5G/satellite. Serial is legacy
- **PDUs (Protocol Data Units)** — name for data at each layer: L7=Data, L4=Segment, L3=Packet, L2=Frame, L1=Bits
- MTU (Maximum Transmission Unit) — Ethernet max frame size is 1500 bytes. Larger data must be broken into segments at L4 before transmission
- Encapsulation — headers are added at each layer going down the stack. Decapsulation — headers are stripped at each layer going up the stack on the receiving side
- Each layer communicates horizontally with its peer layer on the other device using the same protocol (HTTP↔HTTP, TCP↔TCP, IPv4↔IPv4, Ethernet↔Ethernet)
- The OS (Windows, Android, iOS) handles all encapsulation/decapsulation transparently — users just open a browser
- **EtherType field** in the Ethernet II frame connects L2 to L3 — tells the NIC which L3 protocol to use. Key values (hexadecimal): 0x0800=IPv4, 0x0806=ARP, 0x86DD=IPv6, 0x2000=CDP
- **IANA (Internet Assigned Numbers Authority)** maintains the official registry of protocol numbers, port numbers, and EtherTypes. Key IP protocol numbers: 1=ICMP, 6=TCP, 17=UDP, 89=OSPF
- **IP Protocol number field** in the IP header connects L3 to L4 — tells L3 whether to hand the segment to TCP or UDP
- **Port numbers** at L4 connect transport to application — the protocol lives at L7, but the port number at L4 routes the data to the right app on the receiving device
- Three port number ranges: 0–1023 = System/Well-known ports, 1024–49151 = User/Registered ports, 49152–65535 = Dynamic/Ephemeral ports (randomly assigned per session by the OS)
- Ephemeral ports explained — when you open Chrome and Firefox both using HTTP, each session gets a different random source port so return traffic goes to the right browser window
- When server replies, source/destination MAC, IP, and port numbers all swap — sender becomes receiver and vice versa
- Full identifier chain connecting layers: EtherType (L2→L3) → Protocol number (L3→L4) → Port number (L4→App)
- Packet Tracer simulation mode — filter for specific protocols (HTTP, ARP), step through traffic frame by frame, inspect full PDU contents at each layer including EtherType, protocol number, and port numbers in live packets
