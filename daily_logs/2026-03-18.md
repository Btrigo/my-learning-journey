## 🧠 What I Learned / Key Takeaways
- AWS Services scope:
    - global — IAM, Route 53, CloudFront, STS, WAF
    - zonal — EC2 instances, EBS volumes (EBS snapshots = regional)
    - regional = everything else (default assumption)
- Support Plans — key cutoffs:
    - Basic + Developer = 7 core Trusted Advisor checks only, no Support API
    - Business = first tier with full Trusted Advisor, Support API, TAM pool, <1hr critical response
    - Enterprise = dedicated TAM, <15 min critical response
- Load Balancer types by OSI layer:
    - ALB (Layer 7) = HTTP/HTTPS, path-based routing, host-based routing, WebSockets, microservices
    - NLB (Layer 4) = TCP/UDP, extreme performance, millions of req/sec, low latency
    - GLB (Layer 3/4) = virtual appliances, firewalls, security inspection
- On-premises deployment = AWS Systems Manager + AWS CodeDeploy (only two that reach outside AWS)
- IAM Access Analyzer = finds resources shared with external entities, validates policies, generates least-privilege policies from CloudTrail logs
- Programmatic access (CLI/SDK) = Access Keys (Access Key ID + Secret Access Key), NOT username/password
- Console access = username + password + MFA
- ElastiCache = in-memory caching, sits between app and database, reduces DB load (Redis or Memcached)
- AWS Professional Services = AWS sends their own consultants for project-based hands-on help
- Linux networking:
    - NetworkManager managed=false in /etc/NetworkManager/NetworkManager.conf blocks WiFi management
    - iwlwifi disable_11ax=1 in /etc/modprobe.d/iwlwifi.conf disables WiFi 6 on Intel cards
    - dmesg logs show authentication vs association — association failure = router compatibility issue
    - rfkill list / rfkill unblock wifi — checks and clears software radio blocks

## 🛠 Commands / Tools Practiced
- `lspci | grep -i network` — identify network controller
- `lsmod | grep iwlwifi` — check if WiFi driver is loaded
- `sudo dmesg | grep -i iwlwifi` — check kernel messages for WiFi driver errors
- `nmcli device status` — check NetworkManager device states
- `nmcli device wifi list --rescan yes` — force WiFi scan
- `rfkill list` / `rfkill unblock wifi` — check/clear radio blocks
- `sudo ip link set wlo1 up` — bring WiFi interface up manually
- `cat /etc/NetworkManager/NetworkManager.conf` — inspect NM config
- `sudo nano /etc/modprobe.d/iwlwifi.conf` — edit driver options
- `ip link show` — show all network interfaces and states
