## What I Learned / Key Takeaways
- **AWS Budgets** = alerting tool for cost/usage thresholds; **Cost Explorer** = visualization, trend analysis, and forecasting based on historical usage; these are commonly confused on the exam
- **Cost and Usage Report (CUR)** = raw billing data dumped to S3 for deep external analysis — not for alerts or trend visualization
- **Standard Reserved Instances** give the deepest discount but lock you to a specific instance type and region; **Savings Plans** are more flexible (any instance type/region) but slightly less discount — when a question says "willing to commit to specific instance type and region," Standard RIs win
- **Enterprise On-Ramp** is the minimum support plan for TAM access (pool of TAMs); **Enterprise** gives a dedicated TAM — this is a common exam trap
- **Business** is the minimum support plan for full Trusted Advisor checks; Basic and Developer only get core checks
- Support plan critical response times: Business = 1hr, Enterprise On-Ramp = 30min, Enterprise = 15min
- **Serverless** means no server management — you write code, AWS handles infrastructure, scaling, and you pay per execution (Lambda is the canonical example)
- **Vertical scaling** = beefing up one machine (diminishing cost returns, single point of failure); **Horizontal scaling** = cloning across multiple smaller machines (better availability, lower cost)
- **Band steering** — ISP combo boxes broadcast a single SSID across both 2.4GHz and 5GHz bands and decide which band to assign devices; you lose granular control without a dedicated router
- VPCs isolate customer resources from each other within AWS; subnets and security groups control what can communicate internally and externally
