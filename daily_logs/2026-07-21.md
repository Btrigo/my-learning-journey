## 🧠 What I Learned / Key Takeaways

### DevSecOps Fundamentals
- DevSecOps integrates security throughout the entire software development lifecycle instead of treating it as a final checkpoint before deployment.
- Security should be implemented as early as possible ("Shift Left") to reduce risk, lower remediation costs, and improve software quality.
- Successful DevSecOps is built on collaboration, automation, and continuous improvement—not just security tools.

### Software Development Life Cycle (SDLC)
- Learned the seven phases of the SDLC:
  1. Planning
  2. Requirements Definition
  3. Design & Prototyping
  4. Software Development
  5. Testing
  6. Deployment
  7. Operations & Maintenance
- Planning establishes project scope, feasibility, timelines, resources, and overall project goals.
- Requirements Definition gathers functional and non-functional requirements and documents them in a **Software Requirements Specification (SRS)**.
- Design & Prototyping defines application architecture, user interfaces, databases, networking, communication methods, and security controls before development begins.
- Architecture Design Reviews (**ADR**) help ensure multiple engineering teams remain aligned before implementation.
- Software Development focuses on implementing the approved design while following coding standards, documentation, and secure coding practices.
- Testing includes:
  - Test case design
  - Test environment creation
  - Functional testing
  - Regression testing
  - Automated testing
  - Bug reporting
- Deployment is typically automated and should include rollback capabilities when releases fail.
- Operations & Maintenance focuses on uptime, monitoring, incident response, bug fixes, infrastructure automation, and long-term application stability.

### DevOps Culture
- Learned the **CALMS** framework:
  - **C**ulture
  - **A**utomation
  - **L**ean
  - **M**easurement
  - **S**haring
- DevOps emphasizes collaboration between Development and Operations while encouraging automation and continuous feedback.

### DevOps Metrics
- Learned about the four primary **DORA metrics**:
  - Lead Time
  - Deployment Frequency
  - Mean Time to Recovery (MTTR)
  - Change Failure Rate (CFR)
- These metrics help engineering organizations evaluate software delivery performance and identify opportunities for improvement.
