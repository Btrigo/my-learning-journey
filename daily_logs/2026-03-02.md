## 🧠 What I Learned / Key Takeaways

- **AWS AI/ML stack has 3 tiers:** Pre-built AI Services (no expertise needed) → SageMaker AI (custom models, managed infra) → ML Frameworks & Infrastructure (full control, DIY)
- **ML workflow:** Training (historical data → model) → Inference (new data → predictions). These are distinct phases with distinct purposes.
- **AI hierarchy:** AI ⊃ ML ⊃ Deep Learning ⊃ Generative AI — each is a subset of the one above
- **Foundation Models (FMs)** differ from traditional ML models in a key way: traditional models are trained for one task; FMs are pre-trained on massive data and adaptable to *multiple* tasks
- **Polly vs Textract name trap:** Polly = text-to-speech; Textract = extract text from documents. Got this wrong on the course knowledge check — important to lock in
- **Service mapping practice:** Reinforced Comprehend (sentiment), Lex (chatbots/conversational), Rekognition (images/video), Transcribe (speech→text), Personalize (recommendations)
- **Module 7 miss worth remembering:** Aurora is a *relational* database — it handles structured data only. Not a fit for unstructured data or real-time streaming use cases
