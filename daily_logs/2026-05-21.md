## 📚 What I Learned / Key Takeaways
- **What IaC actually is**: Infrastructure defined as code — Terraform reads your `.tf` files and makes the necessary AWS API calls to build exactly what you described. Reproducible, version-controlled, and deployable with one command.
- **Terraform vs CI/CD vs Automation**: Terraform provisions infrastructure. CI/CD automates deployment pipelines. They are separate concerns. `terraform apply` is a form of automation but not CI/CD — that requires a trigger (e.g. GitHub Actions firing on a git push).
- **Three core Terraform files**:
  - `main.tf` — resource definitions (the actual infrastructure)
  - `variables.tf` — input variables (region, bucket name, etc.)
  - `outputs.tf` — values printed after apply (CloudFront URL, bucket name)
- **Terraform workflow**: `init` → `plan` → `apply` → `destroy`
  - `init` downloads the provider plugin, sets up local backend
  - `plan` shows exactly what will be created/changed/destroyed without doing it
  - `apply` executes the plan and builds the infrastructure
  - `destroy` tears everything down cleanly
- **HCL vs JSON**: HCL (HashiCorp Configuration Language) is Terraform's native language. S3 bucket policies must be submitted as JSON — `jsonencode()` converts HCL objects to JSON automatically.
- **Resource references**: Terraform resources reference each other using `resource_type.resource_name.attribute` syntax (e.g. `aws_s3_bucket.portfolio.id`). Terraform figures out the dependency order automatically.
- **`(known after apply)`**: Values Terraform can't know until the resource is actually built (ARNs, IDs, domain names). They get filled in at apply time and printed via outputs.
- **`terraform.tfstate`**: Terraform's state file — tracks what it has built in AWS. Never delete this while actively managing infrastructure. Never commit it to GitHub (may contain sensitive values).
- **`.terraform.lock.hcl`**: Locks the provider version for reproducibility. Should be committed to version control.
- **BucketNotEmpty error**: Terraform won't delete a non-empty S3 bucket by default. Must empty it first with `aws s3 rm s3://bucket-name --recursive` or add `force_destroy = true` to the bucket resource.
- **Windows PATH**: When installing tools manually (not via package manager), the executable folder must be added to the system PATH so Windows can find it from any terminal session.
