## 🧠 What I Learned / Key Takeaways
- **Route 53 hosted zone**: A container that holds all DNS records for a domain. Creating one generates 4 NS records (nameservers) that become the authoritative DNS source for the domain once plugged into the registrar.
- **SOA record**: Start of Authority — auto-generated by Route 53, never manually edited. Contains administrative metadata about the zone. Not relevant to day-to-day configuration.
- **DNS record types recap**: A (IPv4), AAAA (IPv6), CNAME (hostname alias, cannot be used on naked/root domain per DNS spec), MX (mail routing), TXT (domain verification/SPF/DKIM/DMARC), NS (nameservers), SOA (auto-generated), ALIAS (Route 53-specific — maps root domain to a hostname by resolving internally to an IP).
- **Registrar vs DNS separation**: Domain registrar holds ownership of the domain. DNS provider answers queries. These are independent — kept registrar at Hostinger, delegated DNS authority to Route 53 by updating nameservers. No domain transfer required.
- **S3 bucket policy structure**: JSON document with Version, Statement array, Sid (label), Principal (who), Effect (Allow/Deny), Action (what), Resource (what ARN), and optional Condition. AWS actions follow `service:Action` naming convention (e.g. `s3:GetObject`).
- **Least privilege in bucket policy**: Granted only `s3:GetObject` to CloudFront — read only. No write, delete, or admin actions. Scoped Resource to `arn:aws:s3:::brandontrigo.com/*` (all objects, not the bucket itself). Added Condition locking access to specific CloudFront distribution ARN — prevents any other CloudFront distribution from reading the bucket.
- **ACM DNS validation**: ACM adds CNAME records to Route 53 to prove domain ownership. Certificate stays in Pending validation until those records propagate and ACM can resolve them. Auto-renews without manual intervention once issued.
- **DNS propagation**: Nameserver changes don't take effect instantly — old nameservers are cached across global DNS resolvers with TTL values. Can take minutes to hours for full propagation. Can monitor progress at dnschecker.org using NS record type.
- **S3 website endpoint vs bucket endpoint**: Must use website endpoint as CloudFront origin for static hosting — bucket endpoint doesn't understand index documents or static site routing.
