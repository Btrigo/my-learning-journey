## 🧠 What I Learned / Key Takeaways

### AWS DataSync
- Managed service for automating large-scale data transfers between on-premises storage and AWS, or between AWS storage services
- Requires an agent VM on-premises for on-prem transfers; no agent needed for AWS-to-AWS transfers
- Handles checksums, encryption in transit, scheduling, and monitoring automatically
- DataSync = network-based automated transfer | Storage Gateway = ongoing hybrid access | Snow Family = petabyte-scale physical transfer

### Amazon Connect vs AWS Direct Connect
- Amazon Connect = cloud-based contact center (voice + chat, Lex/Comprehend/Transcribe integrations, pay-per-use)
- AWS Direct Connect = dedicated private network connection to AWS, bypasses the public internet
- Classic naming trap — completely different services

### Origin Access Control (OAC)
- Restricts S3 so it only accepts requests signed by your CloudFront distribution (SigV4)
- Replaces the legacy OAI — OAC supports SSE-KMS, all S3 regions, all HTTP methods
- Exam signal: "restrict S3 access to CloudFront only" → OAC

### S3 Storage Classes — Key Scenarios
- Frequently accessed, no retrieval charges → S3 Standard (no retrieval fee at all)
- Infrequent access + re-creatable + immediate retrieval → S3 One Zone-IA (cheapest appropriate option)
- S3 pricing = storage + requests + data transfer OUT; data transfer IN is always free
- Lifecycle policy transitions incur a per-object fee — setting up the policy itself is free
- Bucket policies = pure access control, zero cost

### EC2 Savings Plans vs Reserved Instances
- Compute Savings Plans = maximum flexibility (any region, family, size, OS, tenancy + Lambda + Fargate), up to 66% off
- EC2 Instance Savings Plans = locked to instance family + region, up to 72% off
- Exam signal: "regardless of region, instance family, size, OS, or tenancy" → Compute Savings Plans

### AWS Support Plans (Classic Tiers)
- API access starts at Business — Basic and Developer have no Support API
- Concierge Support (billing/account focus) available at Enterprise On-Ramp and Enterprise
- TAM: Enterprise On-Ramp = pool of TAMs; Enterprise = dedicated TAM

| Plan | API | Concierge | TAM | Urgent Response |
|---|---|---|---|---|
| Basic | ❌ | ❌ | ❌ | No cases |
| Developer | ❌ | ❌ | ❌ | 12 hrs (business hours) |
| Business | ✅ | ❌ | ❌ | 1 hr |
| Enterprise On-Ramp | ✅ | ✅ | ✅ (pool) | 30 min |
| Enterprise | ✅ | ✅ | ✅ (dedicated) | 15 min |

> Note: AWS announced in early 2026 that Business Support and Enterprise On-Ramp are being retired by Jan 1, 2027 in favor of Business Support+, Enterprise Support, and Unified Operations. Exam as of April 2026 still likely tests classic tier names.

### Global Accelerator vs CloudFront
- Global Accelerator = routes traffic over AWS private backbone, static anycast IPs, TCP/UDP, multi-region failover
- CloudFront = caches content at the edge, HTTP/HTTPS
- Signals for Global Accelerator: static IPs, TCP/UDP, non-HTTP, gaming/VoIP/IoT

### Redshift Spectrum vs Athena
- Both query S3 directly without loading data first; both use Glue Data Catalog for schema
- Athena = standalone serverless, no infrastructure, pay per TB scanned, best for ad hoc
- Redshift Spectrum = requires existing Redshift cluster, best for joining warehouse + S3 data

### AWS Systems Manager (SSM)
- Session Manager = browser shell to EC2, no SSH, no port 22, no bastion host, built-in audit logs
- Run Command = execute scripts across a fleet simultaneously
- Patch Manager = automated OS patching on schedule
- Parameter Store = free config/secrets key-value store; vs Secrets Manager (paid, auto rotation)

### Service Control Policies (SCPs)
- AWS Organizations feature — sets maximum permissions boundary for entire accounts
- SCPs restrict, they don't grant; blocks even full admins
- SCP = the ceiling. IAM = what's allowed under that ceiling.
- Signals: multiple accounts, AWS Organizations, restrict account-level actions → SCP

### AWS Cost Management Tools
- Cost Explorer = visualize and analyze spending trends ("rearview mirror")
- AWS Budgets = alert when cost/usage threshold is hit or forecasted ("guardrail")
- Cost & Usage Report (CUR) = most granular line-item data, queryable via Athena ("raw data dump")
- Signal: "most detailed cost data / query with Athena" → CUR | "alert when exceeded" → Budgets

### Volume / Tiered Pricing
- S3 (storage per GB) and data transfer OUT from EC2 have tiered volume pricing
- Data transfer IN is always free
- Consolidated Billing via AWS Organizations pools usage across accounts to hit discount tiers faster

### AWS Lightsail
- IaaS, not SaaS — simplified VPS with flat-rate bundled pricing (compute, storage, DNS, transfer)
- EC2 with training wheels; for developers who don't need full EC2/VPC complexity

### AWS Wavelength
- Embeds AWS compute inside telecom 5G networks — eliminates round trip to AWS region
- Use cases: AR/VR, game streaming, real-time ML inference, connected vehicles, IoT
- Signals: "5G", "telecom", "single-digit millisecond latency for mobile" → Wavelength

### Serverless Services (CLF-C02 Relevant)
- Compute: Lambda, Fargate
- Storage: S3, DynamoDB, Aurora Serverless
- Integration/Messaging: SQS, SNS, EventBridge, Step Functions, API Gateway
- Analytics: Athena, Kinesis, Glue
- Other: Cognito, AppSync
- Classic serverless trio: Lambda + API Gateway + DynamoDB

### Autoscaling on AWS
- EC2 Auto Scaling = core service; ECS/Fargate, Lambda, DynamoDB, Aurora Serverless all autoscale natively
- AWS Auto Scaling (the service) = centralized scaling policies across multiple services from one interface
- Always paired with ELB — ELB distributes traffic, Auto Scaling adjusts capacity
- Autoscaling is the mechanism behind the cloud advantage of elasticity

### AWS IoT Greengrass
- Runs Lambda functions locally on the IoT device itself — no round trip to AWS
- Operates offline, syncs when connectivity restores; manages fleet software deployments remotely
- IoT Core = cloud HQ; Greengrass = mini AWS brain on the device
- Signals: "process IoT data locally", "intermittent connectivity", "run Lambda on IoT hardware" → Greengrass

### EC2 Instance Store
- Temporary block storage physically attached to the host — lost on stop/hibernate/terminate
- Very high performance, included in instance price, only on certain instance types
- Good for: scratch space, caches, buffers. Bad for: databases, anything persistent.
- Keyword: "ephemeral storage" → instance store | EBS = persistent

### EBS Boot Volumes
- SSDs can be boot volumes: gp2/gp3 ✅ and io1/io2 ✅
- HDDs cannot: st1 ❌ and sc1 ❌
- Rule: SSDs boot, HDDs don't

### Amazon Aurora
- Self-healing storage — automatically scans and repairs corrupted data in the background
- 6 copies across 3 AZs; survives loss of 2 copies (writes) or 3 copies (reads)
- Up to 5x faster than MySQL, 3x faster than PostgreSQL; up to 15 read replicas
- Has serverless option (Aurora Serverless)

### Amazon Kendra
- ML-powered enterprise search — returns specific answers to natural language questions, not just links
- Connects to S3, SharePoint, Salesforce, RDS, and more; indexes all sources
- Signal: "intelligent enterprise search", "search internal documents with natural language" → Kendra

**ML service quick reference:**

| Service | Does |
|---|---|
| Kendra | Enterprise document search |
| Lex | Conversational chatbots |
| Comprehend | NLP, sentiment analysis |
| Rekognition | Image/video analysis |
| Textract | Extract text from documents |
| Transcribe | Speech to text |
| Polly | Text to speech |

### Well-Architected Framework — Operational Excellence Pillar
- "Make frequent, small, reversible changes" lives in Operational Excellence — not Reliability
- Other principles: operations as code, refine procedures frequently, anticipate failure, learn from all failures
- Reliability = recovering from change; Operational Excellence = making the change carefully

**All 6 pillars:**

| Pillar | Core idea |
|---|---|
| Operational Excellence | Run and improve systems, small changes, learn from failure |
| Security | Protect data and systems |
| Reliability | Recover from failure, meet demand |
| Performance Efficiency | Use resources efficiently |
| Cost Optimization | Eliminate waste, right-size |
| Sustainability | Minimize environmental impact |

### AWS Transit Gateway
- Hub-and-spoke network hub connecting many VPCs and on-premises networks at scale
- Replaces complex VPC peering meshes; supports multi-account via RAM, multiple route tables, multicast
- Signal: "connect many VPCs at scale" or "simplify network across accounts" → Transit Gateway

### AWS Trusted Advisor
- Proactive best practice recommendations across: Cost Optimization, Performance, Security, Fault Tolerance, Service Limits
- Signal: "conform to AWS best practices" → Trusted Advisor (not Config, not CloudWatch)
- Config = tracks config changes and compliance over time; Trusted Advisor = proactive recommendations

  - Agile, continuous improvement, culture change → People
