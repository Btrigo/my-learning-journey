## ðŸ§  What I Learned / Key Takeaways

**Networking & Security:**
- Security groups act as stateful virtual firewalls at the instance level
- Distinction between exposing SSH (admin access, lock down tight) vs HTTP (public service, calculated risk)
- Understanding when exposing ports is a security risk vs. necessary functionality
- Public web servers require HTTP (80) open to 0.0.0.0/0 by design
- SSH from anywhere (0.0.0.0/0) is dangerous; SSH from specific IP is secure

**RSA Key Pairs:**
- How asymmetric encryption works for SSH authentication
- Public key stored on server, private key stays on client
- Challenge-response authentication prevents private key from traveling over network
- Key permissions must be 400 (read-only for owner) for SSH to accept them
- Default usernames vary by AMI (ec2-user for Amazon Linux, ubuntu for Ubuntu)

**Apache Security Hardening:**
- Directory indexing (`Options -Indexes`) prevents file enumeration attacks
- `ServerTokens Prod` hides version details in HTTP headers (reduces reconnaissance)
- `ServerSignature Off` removes version info from error pages
- Attack surface reduction through minimal information disclosure
- Defense-in-depth: layering multiple security controls

**Cloud Concepts:**
- VPCs are free logical network boundaries (don't "run", just exist)
- Default VPC automatically created in each region
- Shared Responsibility Model: AWS secures infrastructure, I secure what I deploy
- Even servers without sensitive data are valuable targets (computing resources, network position)
- Stopped instances cost $0 but attached EBS volumes cost ~$0.08/GB-month

**AWS Resource Management:**
- Difference between terminated and stopped instances
- EBS volumes can persist after instance termination (orphaned volumes cost money)
- Security groups are reusable and cost nothing
- Key pairs stored in AWS are just metadata (free)
- Proper cleanup prevents unexpected charges

**Package Management:**
- Amazon Linux uses `yum` (RPM-based, Red Hat family)
- Ubuntu uses `apt` (Debian-based)
- Apache package called `httpd` on RHEL systems, `apache2` on Debian systems
- `systemctl` manages services (start, stop, enable, status)

**Text Editors:**
- Nano is beginner-friendly (Ctrl+X to exit, shows commands at bottom)
- Vim is powerful but has steep learning curve (`:wq` to save and quit)
- Both accomplish same task (editing config files)
- Chose nano for accessibility during learning phase

**Documentation:**
- Importance of documenting security controls with "why" not just "what"
- Professional README structure includes architecture, security analysis, cleanup procedures
- Screenshot sanitization critical (redact IPs, instance IDs, account numbers)
- Portfolio-ready documentation explains security thinking, not just commands
