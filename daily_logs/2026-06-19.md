## 🧠 What I Learned / Key Takeaways
- Horizontal scaling beats vertical scaling on both cost (vertical has diminishing returns) and stability (no single point of failure)
- Load balancers distribute traffic via round robin, least connections, or least utilization
- "Serverless" has two meanings now — original (Lambda: true pay-per-execution) vs. newer AWS usage (still paying for underlying instances, just not managing them)
- Event-driven architecture (pub/sub via SNS/EventBridge) decouples services compared to tightly-coupled request/response
- Availability (uptime, via AZs + load balancing) and durability (data survival via redundant copies) are related but distinct
- Git Bash (MINGW64) auto-converts Unix-style paths (`/index.html`) into Windows paths, silently breaking AWS CLI commands — PowerShell doesn't have this issue
