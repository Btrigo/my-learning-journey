## üß† What I Learned / Key Takeaways
- **Elastic IPs** provide a static public IPv4 address that can be reassociated with another instance, allowing endpoint stability even if infrastructure changes.
- **Launch Templates** standardize EC2 configuration (AMI, instance type, security groups, etc.) and are foundational for scalable, repeatable deployments.
- **Tenancy** determines whether compute runs on shared hardware or dedicated physical hardware ‚Äî typically relevant for compliance or licensing requirements.
- **Dedicated Instances and Dedicated Hosts** are used when regulatory, contractual, or licensing constraints require physical isolation.
- **Reserved Instances vs Convertible RIs** differ primarily in flexibility ‚Äî Standard offers higher savings, Convertible offers exchange flexibility.
- **Spot Instances** provide significant cost savings for fault-tolerant and flexible workloads.
- **Savings Plans** reduce costs through usage commitments rather than reserving specific instance attributes.
- **Single Sign-On (SSO)** centralizes authentication, allowing access to multiple systems after a single login.
- In a **Zero Trust model**, identity becomes the primary security boundary rather than the network perimeter.
- **CloudTrail** records API activity for auditing; **CloudWatch** monitors metrics, logs, and operational health.
