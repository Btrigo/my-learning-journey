## 🧠 What I Learned / Key Takeaways
- **Final score: 84% (55/65)** — passing threshold is 70%. Domain breakdown: Security & Compliance 100%, Technology 100%, Billing & Pricing 87%, Cloud Technology & Services 81%, Cloud Concepts 77%
- **CloudWatch = repository for metrics and logs** (performance monitoring, alarms). **CloudTrail = audit service recording API calls and account activity** (who did what, when)
- **ALB vs NLB**: ALB operates at Layer 7 — supports path-based routing, host-based routing, WebSockets, HTTP/HTTPS. NLB operates at Layer 4 — TCP/UDP/TLS, extreme performance, static IP, preserves source IP. Gateway LB = security appliances/firewalls only
- **Layer 4 means visibility from Layer 4 down to Layer 1** — NLB sees IP addresses and ports but cannot read HTTP request content (Layer 7)
- **CAF Security perspective** = the answer when a question mentions "secure" as a primary concern, even in a migration context. Platform perspective = building/provisioning infrastructure
- **Aurora supports MySQL and PostgreSQL only** — never Microsoft SQL Server or Oracle. For SQL Server → RDS or EC2
- **Spot Instances = unused AWS capacity**, not customer capacity. AWS sells spare EC2 capacity at up to 90% discount. Can be reclaimed with 2 minutes notice
- **Convertible RI** = allows changes to instance family, OS, and tenancy during the term. Standard RI = locked in, biggest discount. The keyword "change" → Convertible every time
- **Access Keys** = the credential type for CLI and programmatic access. IAM Users have them. IAM Roles do NOT have long-term access keys — they use temporary credentials via STS
- **Gateway Endpoint** = S3 and DynamoDB only, free. Interface Endpoint (PrivateLink) = all other AWS services, has a cost
- **CloudFront origins** = S3, EC2, ALB, or your own web server. EFS and Glacier cannot be CloudFront origins
- **Trusted Advisor 5 categories**: Cost Optimization, Performance, Security, Fault Tolerance, Service Limits. Mnemonic: "Can Pretty Seals Find Sardines?"
- **Global services**: IAM, Route 53, CloudFront, STS, WAF. Everything else defaults to regional. "If in doubt, it's regional"
- **AWS Elastic Disaster Recovery** = continuously replicates entire server environments (bare metal, VMs, cloud) to AWS for rapid failover with minimum data loss. DMS = database migration tool only, not DR
- **CodeDeploy and Systems Manager** = the two services that work across both AWS and on-premises hybrid environments
- **AWS Professional Services** = paid consulting engagements for specific business outcomes in specialty practice areas. TAM = ongoing advisor included with Enterprise Support. Concierge = billing/account specialists included with Enterprise Support
- **DAX** = caching for DynamoDB only. **ElastiCache** = caching for SQL databases and general application use
- **Snowball** = physical device for large data migrations too big for internet transfer (up to 80TB). Snowmobile = 45-foot shipping container for exabyte-scale migrations
- **Athena** = serverless SQL queries directly against S3 data, pay per query ($5/TB scanned). No servers, no loading data
- **AWS Control Tower landing zone architecture**: S3 manifest → CodePipeline → Organizations + SSO → three specialized accounts (Shared Services, Log Archive, Security). Diagram saved to `images/control-tower-landing-zone.png`
