## 🧠 What I Learned / Key Takeaways
- Loopback ping (127.0.0.1) only tests the OS network stack — never touches physical NIC hardware
- An IP starting with 169.254.x.x means DHCP failed and the OS self-assigned an address
- Copying a folder to a destination name that doesn't already exist creates a brand new folder with that name, instead of placing it where intended
- The root-level home-equivalent directory is owned by root — regular users can't write into it without elevated privileges
- journalctl uses `-n` (not `tail`) to limit output to the most recent lines; regular users see a restricted view of logs unless added to specific groups
- A practice platform's networking content overlaps usefully with troubleshooting fundamentals, but its advanced web-attack content is pentest-specific and not relevant to current goals
