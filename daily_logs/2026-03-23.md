## 🧠 Key Takeaways

### Global / Regional / Zonal Services
| Scope | Services | Rule |
|---|---|---|
| Global | IAM, Route 53, CloudFront, WAF, Shield | Identity + DNS + edge delivery |
| Zonal (AZ) | EC2, EBS, Subnets | Physically attached to hardware |
| Regional | Everything else (S3, RDS, Lambda, EFS, VPC...) | Default assumption — if in doubt, it's regional |

---

### Serverless vs Managed — Critical Distinction
| Type | Meaning | Examples |
|---|---|---|
| Serverless | No infrastructure to provision at all — AWS handles everything | Lambda, Lambda@Edge, Fargate, API Gateway, DynamoDB, S3 |
| Managed | AWS handles operations but you still provision infrastructure | RDS, ElastiCache, EMR, Redshift |

---

### AWS Database Map
| Use Case | Service |
|---|---|
| JSON / key-value / NoSQL | Amazon DynamoDB |
| Relational / SQL (MySQL, PostgreSQL) | Amazon RDS or Aurora |
| Microsoft SQL Server or Oracle | Amazon RDS only — NOT Aurora |
| Data warehouse / business intelligence / petabyte analytics | Amazon Redshift |
| In-memory caching | Amazon ElastiCache |
| Search and analytics | Amazon OpenSearch |
| Ledger database | Amazon QLDB |
| Graph database | Amazon Neptune |

> ⚠️ **Aurora Trap:** Aurora supports MySQL and PostgreSQL ONLY. If the question names SQL Server or Oracle → RDS, every time.

### Security Service Differentiation
| Service | What It Does | Trigger Words |
|---|---|---|
| Amazon Macie | ML-powered discovery and protection of sensitive data (PII, IP) in S3 | PII, sensitive data, intellectual property, S3 data, data classification |
| Amazon GuardDuty | ML-powered threat detection — monitors for malicious activity across your AWS account | malicious activity, unauthorized behavior, threat detection, account compromise |

> 💡 **Memory Hook:** Macie = **M** for "my data" (S3 buckets). GuardDuty = guard watching the door (account behavior).

### Migration Service Differentiation
| Service | What It Does | When to Use |
|---|---|---|
| AWS DMS | Migrates actual data from one database to another — source stays live | Migrating database to AWS (same or different engine) |
| AWS SCT | Converts database schema when switching engines | Different engine migration ONLY (e.g. Oracle → Aurora PostgreSQL). Use SCT first, then DMS |
| AWS Glue | ETL service — extracts, transforms, loads data for analytics pipelines | Data pipelines and analytics, NOT database migration |
| AWS Application Migration Service | Migrates entire servers/applications to AWS | Lift-and-shift of servers, not just databases |

### EC2 Reserved Instances — Standard vs Convertible
| Characteristic | Standard RI | Convertible RI |
|---|---|---|
| Discount (1yr) | ~40% | ~31% |
| Discount (3yr) | ~60% | ~54% |
| Change instance family/OS/tenancy/payment | ❌ No | ✅ Yes |
| Exchange flexibility | Limited (AZ, size, networking) | Equal or greater value only — never downgrade |
| Best for | Steady-state, predictable workloads | Workloads that may change over time |

> 💡 **Key Rule:** Convertible RI exchanges go UP or SIDEWAYS — never down. "Fraction of a day/week/month" describes **Scheduled RIs**, not Convertible.

---

### EC2 Instance Metadata vs User Data
| Concept | What It Is |
|---|---|
| Instance Metadata | Data ABOUT the instance — ID, IAM profile, kernel info. Accessed at `http://169.254.169.254/latest/meta-data/` from within the instance |
| Instance User Data | Scripts passed at launch to run on startup (install commands, config) |
| AMI | Template used to launch the instance — not a runtime data source |
| Resource Tag | Key-value labels for organizing/billing — not instance self-info |

### Amazon Redshift
- **Official definition:** "A fully managed, petabyte-scale data warehouse service in the cloud"
- Uses **columnar storage** — optimized for analytics, not transactions
- **Massively Parallel Processing (MPP)** — queries run across multiple nodes simultaneously
- **Redshift Spectrum** — query data directly in S3 without loading it into Redshift first
- Uses standard SQL — compatible with existing BI tools (QuickSight, Tableau, Power BI)
- ML is used internally for performance optimization — does NOT make it an ML service
- Trigger words: data warehouse, business intelligence, petabyte, analytical queries, columnar storage

> ⚠️ **Exam Trap:** IAM and S3 are NOT on the permitted list — don't be fooled by their security relevance.

### S3 Lifecycle Policy
- **S3 Lifecycle Policy** → automatically transitions objects to cheaper storage classes based on age/access patterns
- **AWS Transfer Family** → transfers files into/out of S3 using SFTP/FTP/FTPS — NOT a storage tiering tool
- **AWS Storage Gateway** → connects on-premises environments to AWS storage — NOT S3 tiering
- Trigger phrase: "automatically move infrequently accessed data to cheaper storage" → **S3 Lifecycle Policy**

---

### RDP = EC2 Rule
- RDP mentioned in a question → **EC2 on Windows**, never RDS
- DBA needs OS/DB level access → **EC2**, not RDS (RDS is fully managed, no OS access)
- SQL Server bundled AMI on EC2 = license included, no separate purchase needed
