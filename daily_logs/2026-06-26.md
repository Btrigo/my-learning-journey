## 🧠 What I Learned / Key Takeaways

- Terraform and AWS CDK solve many of the same infrastructure problems but approach them differently. Terraform uses HCL, while CDK allows infrastructure to be defined using general-purpose programming languages like Python.
- My biggest learning bottleneck is no longer understanding AWS concepts—it's becoming comfortable writing real Python while continuing to build cloud infrastructure.
- AWS CDK provides an opportunity to learn AWS architecture, Infrastructure as Code, and Python simultaneously.
- Python is more than just another syntax to learn; it opens the door to AWS CDK, boto3, Lambda development, automation scripts, and many other cloud engineering tasks.
- Infrastructure should always be understood manually before being automated. Building resources in the AWS Console first helps reinforce why each service exists before recreating everything in code.
- Automation is becoming a major long-term focus. Rather than simply deploying infrastructure, I want to begin thinking about operational tasks that can eventually be automated using Python and boto3.
