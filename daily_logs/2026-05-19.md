## 📚 What I Learned / Key Takeaways
- **Route 53 as registrar vs DNS**: Route 53 can serve as both the domain registrar AND DNS provider, eliminating the need for a third party like Hostinger. Current live site uses Hostinger as registrar with nameservers pointed at Route 53 — the new build would cut Hostinger out entirely.
- **Custom domain is not required for a CloudFront stack**: Without a custom domain, Route 53 isn't needed at all. The stack is simply S3 → OAC → CloudFront → CloudFront URL. Free.
- **Rebuilding in AWS doesn't touch the live site**: Every AWS resource is independent. A new S3 bucket and CloudFront distribution have no connection to existing ones unless you deliberately change DNS records.
- **Windows PATH**: When you install a program like Terraform manually (not via a package manager), Windows won't recognize it as a command until you add its folder to the system PATH. PATH is the list of folders Windows scans when you type a command in the terminal.
- **Why Terraform over CloudFormation**: Terraform is cloud-agnostic, widely adopted across the industry, and a stronger resume/portfolio signal for a network cloud engineer path.
