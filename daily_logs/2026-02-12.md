## üß† What I Learned / Key Takeaways
- **Networking flow for web apps**: User ‚Üí Frontend (S3/CloudFront) ‚Üí API Gateway ‚Üí Backend (Lambda/EC2) ‚Üí Database (DynamoDB/RDS) ‚Üí S3 (storage)
- **Frontend-backend communication**: React makes HTTP requests (fetch/axios) to API endpoints, backend responds with JSON data
- **CORS configuration**: Required to allow frontend and backend on different domains to communicate
- **Authentication flow**: User signs in ‚Üí receives JWT token ‚Üí token sent in Authorization header for all subsequent requests
- **Translation layers**: 
  - AWS DynamoDB Document Client translates between JavaScript objects and DynamoDB's native format
  - ORMs translate between code objects and SQL database tables
  - API Gateway can transform request/response formats
- **File upload optimization**: Use presigned S3 URLs to upload directly to S3 (bypasses backend bottleneck)
- **AWS Amplify vs Manual setup**:
  - Amplify is "all-in-one" framework that abstracts AWS complexity (fast prototyping)
  - Manual setup teaches individual AWS services deeply (better for certification prep)
  - Best approach: Start with Amplify to see the whole picture, then rebuild manually to understand the internals
