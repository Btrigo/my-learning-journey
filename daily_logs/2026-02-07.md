## üß† What I Learned / Key Takeaways
- **Security Groups** act as a firewall at the *instance* level  
  - **YOU CREATE ONLY ALLOW RULES**
  - All other traffic is implicitly denied
- **Network ACLs (NACLs)** act as a firewall at the *subnet* level  
  - Support explicit **ALLOW and DENY** rules  
  - Rules are evaluated in numerical order
- **Public vs Private Subnets** are determined by **route table associations**, not by name
- Internet access requires:
  - An **Internet Gateway**
  - A route (`0.0.0.0/0`) pointing to that gateway
- **Elastic IPs** provide a static public IPv4 address that can be reassigned
- **SSM Session Manager** can replace SSH key pairs:
  - No inbound ports required
  - Access controlled via IAM
  - No private keys stored locally
- You can attach an **IAM role** to an EC2 instance to enable SSM access
- **User Data scripts** run at instance launch and can automate configuration
- **CloudFront**:
  - Uses **origins** (e.g., S3, ALB)
  - Uses **behaviors** to control caching and routing
- Operational best practice:
  - Reboot instances to confirm configuration persistence
  - Terminate instances and clean up resources after labbing
