### AWS CAF — Full Perspective Breakdown
- **Business** — aligns cloud investments to business outcomes; stakeholders: CEO, COO, CFO, CIO, CTO; trigger: "cloud investments," "business outcomes," "digital transformation goals"
- **People** — company-wide change management, skill gaps, culture shift; stakeholders: CTO, Cloud Directors, COO, Cross-functional Leaders, CIO, Enterprise-wide Leaders; ⚠️ trap: "company-wide change management" = People NOT Business
- **Governance** — orchestrates cloud initiatives, maximizes value, minimizes risk; focus areas: portfolio management, benefits realization, risk management, cloud financial management, data governance; stakeholders: CIO, CFO, COO, CTO, CDO, CRO
- **Platform** — builds and modernizes cloud infrastructure, implements new solutions, migrates workloads; stakeholders: Technology Leaders, Architects, Engineers, CTO; trigger: "implement new solutions," "migrate workloads," "modernize infrastructure"
- **Security** — confidentiality, integrity, availability, visibility, auditability, control; stakeholders: CISO, Internal Audit Leaders, Security Architects, Engineers, CCO; trigger: "secure migration," "security objectives," "auditability and control"
- **Operations** — runs and recovers cloud workloads to agreed service levels; stakeholders: Infrastructure & Operations Leaders, Site Reliability Engineers, IT Service Managers; trigger: "run and operate workloads," "service levels," "day-to-day operations"
- Business capabilities = Business, People, Governance. Technical capabilities = Platform, Security, Operations.
- CAF phases: Envision → Align → Launch → Scale
- Launch phase = delivering pilots **in production** at small controlled scale — not in a sandbox or dev environment

### WAF — 6 General Design Principles
Mnemonic: **"Stop Testing Apes Allowing Data Games"**
1. Stop guessing your capacity needs
2. Test systems at production scale
3. Automate to make architectural experimentation easier
4. Allow for evolutionary architectures
5. Drive architectures using data
6. Improve through game days

- "Intelligently guess capacity needs" is a trap — the word **guess** is the kill shot
- "Stick to one cloud architecture" is wrong — the principle is **evolutionary** architectures
- "Test recovery procedures" is a **Reliability pillar** best practice, not a general design principle

### WAF Pillars — Operational Excellence Design Principles
Mnemonic: **OSRPCS — "Oh Shit, Real Problems Can Suck"**
- Perform operations as code ← exam trigger for Operational Excellence
- Make frequent, small, reversible changes
- Refine operations procedures frequently
- Anticipate failure
- Learn from all operational failures

### Networking — Q50 Correction (Route 53 + VPG for on-premises VPN to VPC)
- **Virtual Private Gateway (VPG)** = AWS-side anchor of a Site-to-Site VPN connection
- **Amazon Route 53** = correct second answer because of the **Route 53 Resolver** feature, which provides recursive DNS for your VPC and on-premises networks over Direct Connect or a VPN — this is what enables hybrid cloud DNS resolution
- VPC Peering = connects two VPCs, not on-premises to VPC
- NAT Gateway = allows private subnets to reach internet, not for VPN
- Egress-Only Internet Gateway = IPv6 outbound only

### EC2 Purchasing Options Comparison
| Option | What it is | Key trigger |
|---|---|---|
| **On-Demand** | Pay by hour/second, no commitment | Unpredictable or short-term workloads |
| **Reserved Instance** | 1 or 3 year commitment, significant discount | Steady-state, predictable workloads |
| **Dedicated Instance** | Dedicated hardware, AWS controls which server | Compliance — no hardware sharing |

### Storage Types
| Type | Service | Use case |
|---|---|---|
| Object | S3 | Flat structure, accessed via URL, scales infinitely |
| Block | EBS | Low latency, single AZ, attached to one EC2 instance |
| File | EFS | Shared file system, multiple EC2 instances, regional |

- EBS = zonal (single AZ)
- EFS = regional (spans all AZs in a region)
- S3 = regional (globally accessible via URL)

### Service Scope — Global vs Regional vs Zonal
**Global:** IAM, Route 53, CloudFront, AWS Organizations, WAF, Shield, Global Accelerator
**Regional (default rule — when in doubt, it's regional):** S3, DynamoDB, Lambda, RDS, Aurora, EFS, SQS, SNS, EventBridge, Fargate, ELB, CloudTrail, CloudWatch, Security Hub, GuardDuty, Inspector, Macie, KMS, Cognito
**Zonal:** EBS, EC2 instances, Subnets, NAT Gateway, RDS single-AZ

### Aurora Engine Limitation ⚠️
- Aurora supports **MySQL and PostgreSQL ONLY**
- Never SQL Server, never Oracle
- RDS supports SQL Server — use RDS when the question specifies Microsoft SQL Server

### ElastiCache
- Fully managed in-memory caching service (Redis and Memcached)
- Stores results of I/O-intensive database queries for low-latency retrieval
- Works with RDS, Aurora, and other SQL databases
- DAX = DynamoDB-only caching — never correct for SQL query caching
- Trigger: "cache database query results," "reduce database load," "in-memory cache," "improve read performance"

### IAM Access Analyzer
- Identifies resources shared with external entities
- Generates least-privilege IAM policies based on actual CloudTrail access history
- Trigger: "shared with external entity," "generate IAM policy based on access activity"

### AWS Amplify
- Tools and services for building full-stack web and mobile apps
- Bundles Cognito (auth), S3 (storage), API Gateway + Lambda (APIs), AppSync (GraphQL)
- Trigger: "quickly build and deploy full-stack web or mobile apps," "frontend developers building on AWS"

### Security Groups — Resources You Can Attach To
- EC2 instances (via ENI), RDS, Lambda (in VPC), ECS tasks, ELB/ALB/NLB, ElastiCache, Redshift
- **Cannot attach to:** S3, CloudFront, DynamoDB, Route 53
- Rule: if it lives inside a VPC and has an ENI, it can have a security group

### Elastic Network Interface (ENI)
- Virtual network card attached to EC2 instances inside a VPC
- Contains: primary private IPv4, secondary IPv4s, Elastic IP, public IPv4, security groups, MAC address
- ENIs are AZ-specific
- Security groups are technically attached at the ENI level, not the instance level

### Ways to Provision AWS Resources
- AWS Management Console, AWS CLI, CloudFormation, SDKs
- AWS Concierge, CodePipeline, and Systems Manager are never provisioning tools

### CAF — ESG Definition
- ESG = Environmental, Social, and Governance
- In CAF context: cloud adoption helps companies hit ESG targets (e.g., reducing data center energy waste)
- Not a deep-dive exam topic — listed as a Business outcome only
