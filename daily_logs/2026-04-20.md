## 🧠 What I Learned / Key Takeaways
- **Scaling:** Vertical = bigger single machine (diminishing cost returns, single point of failure); Horizontal = more smaller machines (better cost, better stability) — horizontal is the cloud-native approach
- **Load balancing** sits in front of horizontally scaled instances and distributes traffic using algorithms (round robin, least connections, resource utilization); health-aware so it skips unhealthy instances
- **Autoscaling** automates adding/removing instances based on metrics (CPU, connections, etc.) — AWS Auto Scaling Groups handle this natively
- **Serverless:** Originally meant Lambda-style — write code, no infra awareness, pay per execution. AWS now applies the term more broadly (e.g. OpenSearch Serverless) where you still pay for underlying instances but don't manage them — worth knowing the distinction for the exam
- **EDA / Pub-Sub:** Publisher fires an event to SNS topic or EventBridge event bus; any number of subscribers receive a copy — decouples producer from consumers. Contrast with request-response (tight coupling, sequential calls). Key terms: publisher, subscriber, fan-out
- **Container orchestration:** ECS and EKS solve the problem of deploying, health-checking, and scaling containers across multiple machines — without it you're managing restarts and deployments manually on EC2
- **Storage types:** Object (S3 — media, JSON, CSVs), Block (EBS — volumes, provisioned capacity), Databases (relational SQL, NoSQL like DynamoDB/OpenSearch, cache like ElastiCache)
- **Availability vs Durability:** Availability = app uptime (AZs, horizontal scaling, load balancing); Durability = data safety (multiple copies across data centers — S3 eleven 9s)
- **IaC:** CloudFormation (declarative, template-based) vs CDK (imperative, full programming language, preferred); Terraform spans multiple cloud providers
- **VPC:** Isolates your resources within AWS from other customers by default; subnets (public vs private), security groups control traffic rules — same mental model as on-prem networks but in the cloud
- **Pricing principles:** Pay as you go (variable workloads), Save when you commit (1 or 3yr = up to 72% off), Pay less by using more (volume discounts) — commit = time, more = volume, don't mix these up
- **Cost drivers:** Compute (billed by time), Storage (EBS = provisioned, S3 = actual usage), Outbound data transfer (inbound free, outbound costs money)
- **Billing tools:** Pricing Calculator (pre-deployment estimate) vs Cost Explorer (historical + forecast) vs Budgets (proactive alerts) vs Organizations (consolidation across accounts)
- **Support plans:** Basic (free) → Developer (email) → Business (phone + full Trusted Advisor) → Enterprise On-Ramp (pool of TAMs, <30min critical) → Enterprise (designated TAM, <15min critical) — full Trusted Advisor starts at Business
- **TAM:** On-Ramp = shared pool; Enterprise = designated — this distinction is a frequent exam trap
- **Marketplace = third-party software only** — native AWS services are not Marketplace products; APN supports partners, not end users
