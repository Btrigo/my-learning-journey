## 🧠 What I Learned / Key Takeaways
- S3 is object storage accessed via API, not a file system — objects live in buckets, not directories; max object size is 5 TB with 11 9s durability
- S3 security defaults to fully private; bucket policies (resource-based) vs IAM policies (identity-based) are two distinct access control mechanisms — both can govern S3 access
- EBS is AZ-scoped block storage for EC2 (persistent), EC2 Instance Store is physically attached and ephemeral — data is gone when instance stops/terminates
- EFS = shared Linux file system (NFS, multi-instance, regional); FSx = specialized managed file systems (Windows → SMB/AD, Lustre → HPC)
- AWS Storage Gateway bridges on-prem to AWS — S3 File Gateway, Volume Gateway (iSCSI/EBS), Tape Gateway (S3 Glacier VTL)
- Database management tiers: EC2 (unmanaged, customer owns everything) → RDS (managed) → Aurora/DynamoDB (fully managed)
- Multi-AZ RDS = high availability and failover only — NOT for performance; Read Replicas = performance scaling for reads
- Aurora vs RDS: Aurora is 5x MySQL / 3x PostgreSQL throughput, auto-scales 10 GB → 128 TB, 6 copies across 3 AZs, 99.99% availability, 15 replicas max
- DynamoDB = serverless, fully managed NoSQL, single-digit millisecond latency, flexible schema (partition key is the only required attribute)
- DAX = DynamoDB-specific cache; ElastiCache = general-purpose cache — don't conflate them on exam questions
- Missed Aurora assessment question: Aurora is relational (structured data only) — unstructured/streaming workloads are not Aurora territory
