## 🧠 What I Learned / Key Takeaways
- **EC2 Purchasing Options:** On-Demand = full flexibility, Reserved = commitment discount, Spot = up to 90% off for interruptible workloads, Dedicated Host = physical server for BYOL/compliance. All Upfront + Standard + 3-year = maximum discount.
- **Global vs Regional vs Zonal services:** IAM, Route 53, CloudFront, WAF, STS = Global. EC2 instances and EBS volumes = Zonal. EBS snapshots = Regional. Everything else defaults to Regional.
- **Minimum vs Best Practice:** Minimum AZs for ALB high availability = 2. Best practice = 3. Watch for the word "minimum" on the exam.
- **EventBridge vs Kinesis vs SQS vs SNS:** EventBridge = event-driven architecture and decoupling. Kinesis = high-volume data streaming. SQS = message queue buffer. SNS = fan-out broadcaster.
- **CAPEX vs OPEX:** On-premises = upfront CAPEX. AWS = variable OPEX. The exam will mix these in answer choices deliberately.
- **Support Plans:** Developer = 7 core Trusted Advisor checks, no Support API. Business = full Trusted Advisor + Support API (minimum for API access). Enterprise On-Ramp = pool of TAMs + Concierge. Enterprise = dedicated TAM.
- **Trusted Advisor 5 Categories:** Cost Optimization, Performance, Security, Fault Tolerance, Service Limits. Mnemonic: COPS + F.
- **Database selection:** Aurora = MySQL/PostgreSQL compatible, cloud-native, OLTP, highly scalable. DynamoDB = NoSQL, non-relational, flexible schema, unpredictable traffic. Redshift = OLAP/data warehousing. ElastiCache = SQL database caching (Redis/Memcached). DAX = DynamoDB caching only.
- **Caching rule:** SQL + caching → ElastiCache. DynamoDB + caching → DAX.
- **CloudWatch vs CloudTrail vs AWS Config:** CloudWatch = performance metrics and alarms. CloudTrail = API activity audit log (who did what). AWS Config = resource configuration state and compliance drift detection.
- **Cost tools:** Cost Explorer = visual trend analysis. Cost and Usage Report = most granular, line-item detail, RI tracking. AWS Budgets = spending alerts. Cost Allocation Tags = categorize costs by team/project/environment.
- **AWS Artifact:** The only place to download AWS compliance documents — SOC reports, ISO certifications, PCI reports, BAA, NDA.
- **Consolidated Billing benefits:** One bill for multiple accounts + volume pricing discounts from pooled usage across all accounts. Management account pays for all member accounts (not the reverse).
- **APN Partners:** Consulting Partners = design, architect, migrate, manage workloads. Technology Partners = software products that run on AWS. AWS Marketplace = store to buy software, not a consulting channel.
- **VPN connectivity:** Virtual Private Gateway = AWS side of Site-to-Site VPN. Customer Gateway = on-premises side. NAT Gateway = outbound internet for private subnets only, unrelated to VPN.
- **Region selection factors:** Compliance/data sovereignty, latency to users, service availability, pricing.
- **AWS Elastic Disaster Recovery:** Purpose-built for DR — replicates entire environments (bare metal + databases) continuously with point-in-time recovery. DMS = database migration only, not DR.
- **Shared Responsibility Model:** Managed services (RDS, Aurora, DynamoDB, Lambda) = AWS manages the OS. EC2 = customer manages the OS, patches, and security updates.
- **Access keys:** Used for CLI and SDK (programmatic access). Username/password = Console login only. IAM roles issue temporary credentials automatically — no long-term access keys.
- **CloudFront + S3:** Standard combination for serving static content globally with low latency.
- **AWS Batch:** Fully managed batch processing — runs containerized jobs across AZs, supports Spot Instances, regional service.
