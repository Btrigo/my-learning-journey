## 🧠 What I Learned / Key Takeaways
- POST (Power-On Self-Test) checks CPU, RAM, GPU, and I/O in sequence before OS load; beep codes indicate where it failed
- 169.254.x.x = APIPA address, means DHCP failed — different fix path than a normal IP with DNS/gateway issues
- ipconfig /flushdns clears the local DNS resolver cache — useful when DHCP is fine but sites won't load
- Task Manager's CPU % is total across all cores — a single-threaded process can show "100%" while only maxing one core
- Corrected my own note on SolarWinds experience: it was environmental/physical monitoring (temps, mechanical warnings) escalated to NOC, not network/DHCP monitoring
