### Amazon Cognito
- Two components: **User Pools** (authentication — who you are, returns JWTs) and **Identity Pools** (authorization — temporary AWS credentials via STS)
- User Pools = login system; Identity Pools = key to AWS resources
- Identity Pools support unauthenticated (guest) access; User Pools do not

### GuardDuty vs. Inspector
- **GuardDuty** = active threat detection using ML + behavioral analysis; analyzes VPC Flow Logs, CloudTrail, DNS logs; one-click setup, no agent required; regional service
- **Inspector** = vulnerability assessment (CVEs, misconfigs) on EC2, ECR, Lambda; requires SSM Agent
- GuardDuty = "is someone breaking in?"; Inspector = "are my locks weak?"

### AWS Support Plans
- **Business** = first tier with: 24/7 phone/chat/email, Support API, contextual architecture support, all Trusted Advisor checks
- **Enterprise On-Ramp** = first tier with TAM (shared pool) and Concierge Support Team
- **Enterprise** = dedicated TAM, consultative architecture reviews
- Developer = email only, business hours — no API, no 24/7, no TAM
- "Self-paced labs" is never a Support plan feature — it's a Training/Skill Builder product

### AWS CAF Transformation Phases
- **Envision** = identify transformation opportunities, make the case for cloud (pre-execution)
- **Align** = identify capability gaps, stakeholder alignment, organizational dependencies
- **Launch** = deliver pilot initiatives in production, demonstrate incremental business value
- **Scale** = expand pilots to desired scale, realize and sustain business benefits
- Key distinction: "deliver" = Launch; "expand" = Scale; execution has started = past Envision

### AWS Global Accelerator
- Routes traffic through AWS private backbone network instead of public internet
- Reduces latency for global users; provides static IPs; automatic failover
- vs. CloudFront: CloudFront caches content; Global Accelerator routes traffic (no caching)

### Service Control Policies (SCPs)
- Applied at AWS Organizations level; set maximum permissions for member accounts
- SCPs do NOT apply to the management (root) account of the organization
- Keyword: "control use of services across multiple accounts"

### AWS Systems Manager — Sub-Tools
- Session Manager: SSH-free secure access to EC2
- Patch Manager: automated OS patching at scale
- Run Command: remote scripts/commands across fleet
- Parameter Store: config data and basic secrets (no auto-rotation)
- vs. Secrets Manager: auto-rotation of secrets (DB passwords, API keys)
- vs. CloudWatch: SSM operates on instances; CloudWatch monitors/observes

### IaaS vs PaaS Classification
- **IaaS**: EC2, VPC, EBS (you manage the OS)
- **PaaS**: RDS, ECS, Elastic Beanstalk (AWS manages platform)
- **FaaS**: Lambda
- **SaaS**: WorkMail, Chime, Connect

### AWS Code* Family
- **CodeCommit** = managed Git repo (store)
- **CodeBuild** = compiles code, runs tests, produces artifacts (build)
- **CodeDeploy** = deploys to EC2/Lambda/Fargate (deploy)
- **CodePipeline** = orchestrates the full CI/CD pipeline end-to-end (orchestrate)
- **CodeArtifact** = stores/shares software packages/dependencies (packages)
- CodePipeline is the conductor — it calls the others; it does not build or deploy itself

### S3 Storage Classes
- **Standard**: frequent access, no retrieval fee, ≥3 AZs
- **Standard-IA**: infrequent, millisecond retrieval, per-GB retrieval fee, ≥3 AZs
- **One Zone-IA**: infrequent, millisecond retrieval, per-GB retrieval fee, 1 AZ — use for re-creatable/secondary backup data
- **Intelligent-Tiering**: unknown access patterns, no retrieval fee
- **Glacier Instant**: archive, millisecond retrieval
- **Glacier Flexible**: archive, minutes–hours retrieval
- **Glacier Deep Archive**: archive, 12–48 hours, cheapest overall
- Only Standard and Intelligent-Tiering have **zero retrieval fees**
- "Re-creatable + cheapest + low latency" → One Zone-IA
- "No retrieval fee + frequently accessed" → Standard or Intelligent-Tiering

### Networking Services
- **VPC Peering**: private connection between two VPCs; no transitive routing; no overlapping CIDRs; works across accounts and regions
- **Transit Gateway**: hub-and-spoke for many VPCs; supports transitive routing
- **Site-to-Site VPN**: on-premises → AWS via encrypted tunnel over public internet
- **Direct Connect**: on-premises → AWS via dedicated private fiber; consistent bandwidth
- **Global Accelerator**: routes via AWS backbone; static IPs; non-HTTP traffic optimization

### EC2 / EBS Billing
- EC2 billed per hour or second when **Running**; stops when **Stopped** or **Terminated**
- EBS billed when **Running** AND **Stopped**; only stops when **Terminated**
- "Only EBS billing, no compute billing" = **Stopped** state
- S3 billed **per GB** stored (not per hour, not per file type)
- Data transfer INTO AWS = always free; data transfer OUT = charged per GB

### Other Services Locked In
- **AWS Service Catalog**: portfolio of pre-approved AWS resources; IT governance over what can be deployed
- **AWS IAM Identity Center**: SSO across multiple accounts; "single login" signal
- **AWS Directory Service**: managed Microsoft Active Directory; connects to on-premises AD
- **ElastiCache engines**: Redis (persistence, replication, rich data structures) and Memcached (simple, no persistence) — only two engines
- **S3 Block Public Access**: overrides all other permissions; hardest guarantee against public objects
- **Amazon Macie**: S3-only; ML-based PII/sensitive data discovery and classification
