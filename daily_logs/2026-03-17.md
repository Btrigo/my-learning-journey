## 🧠 What I Learned / Key Takeaways
- **Pricing Calculator vs Cost Explorer**
  - Pricing Calculator → estimate future costs
  - Cost Explorer → analyze past/current usage

- **Consolidated Billing Advantages**
  - Volume discounts across accounts
  - Single bill for entire organization
  - Cost visibility across accounts

- **IAM & Access Keys**
  - Access keys = IAM **User** (long-term credentials)
  - Roles = temporary credentials (STS)

- **Support Plans**
  - Developer = limited features, no Support API, basic Trusted Advisor
  - Business/Enterprise = full features, API access, faster response times

- **Route 53 vs NAT Gateway**
  - Route 53 = DNS resolution (critical for hybrid/VPN setups)
  - NAT Gateway = private subnet → internet only (NOT inter-network communication)

- **CloudWatch vs CloudTrail**
  - CloudWatch = metrics, logs, monitoring
  - CloudTrail = API calls, auditing, “who did what”

- **AWS Professional Services**
  - Paid consulting + implementation (NOT support)
  - Helps build and migrate solutions

## 🔤 AWS Services Starting with “A” (Today’s Focus)
- **Amazon Aurora**
  - MySQL/PostgreSQL-compatible relational database  
  - High performance + high availability (OLTP workloads)

- **Amazon Redshift**
  - Data warehouse for analytics (OLAP, NOT OLTP)

- **Amazon DynamoDB**
  - Fully managed NoSQL key-value database  
  - High scalability, low latency

- **Amazon Timestream**
  - Time-series database (IoT, metrics, time-based data)

## ⚠️ Key Exam Traps / Clarifications
- Access keys → IAM User (NOT roles, groups, or policies)
- NAT Gateway ≠ VPC-to-on-prem or inter-network communication
- Route 53 = DNS resolution (not traffic routing)
- CloudWatch ≠ auditing → CloudTrail handles API tracking
- Professional Services ≠ Support Plans
- “Full access” usually ≠ Developer support plan
