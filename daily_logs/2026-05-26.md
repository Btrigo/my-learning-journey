## 🧠 What I Learned / Key Takeaways
- Networking (the human kind) matters more than certs at entry level — most jobs come from relationships, not resumes. Personality and drive outweigh technical background especially at junior levels
- CCNA is devalued compared to 10-15 years ago due to test dumps and oversaturation — projects, home labs, and volunteering are now the real differentiators
- Don't buy home lab gear without a plan — define objectives first, then buy to match. Packet Tracer covers the majority of CCNA topics and is more powerful than most people give it credit for
- AI in networking is most useful for log analysis and data aggregation, not for blindly troubleshooting — senior engineers who refuse to use it and juniors who over-rely on it are both wrong. The sweet spot is knowing how to frame prompts and verify output
- Return to office = job security for network engineers — physical buildings need IDFs, switches, APs, and ongoing upkeep
- ONT (Optical Network Terminal) converts fiber optic signal to ethernet at the ISP demarc point — it is the handoff between ISP and your router
- Dream Machine Pro Max is an all-in-one gateway/router/firewall/network controller — not a modem, not a Layer 3 switch, but acts as the UniFi ecosystem brain including functioning as a wireless controller for lightweight APs
- SFP+ is a 10Gbps port that accepts transceiver modules (fiber or DAC copper) — used for high-speed uplinks, enables fast NAS file transfers
- Layer 2 switch forwards based on MAC addresses only; Layer 3 switch (aka multilayer switch) adds IP routing capability between subnets/VLANs — same device, two names
- Autonomous APs are self-managed; lightweight APs offload all intelligence to a Wireless LAN Controller (WLC) via CAPWAP tunnels (control + data). WLC enables centralized config, roaming, and RF management across hundreds of APs
- Straight-through cable = same wiring standard (568A or 568B) on both ends, used for connecting different device types (PC→Switch). Crossover = 568A one end, 568B other, used for same device types (Switch→Switch). Auto-MDI/MDIX on modern gear makes this mostly irrelevant in practice but it's tested on CCNA
- Console port (RJ45) and USB console port do the same job — out-of-band local CLI access. RJ45 requires rollover cable + USB-to-serial adapter; USB is direct. Terminal emulator settings: 9600 baud, 8 data bits, no parity, 1 stop bit, no flow control
- MAC address has a BIA (Burned In Address) at the hardware level but can be overridden at the OS level via MAC spoofing — this is why MAC-based port security alone is a weak control
- Edge AI on cameras (G6 PTZ, AI PTZ Industrial) classifies motion events onboard (person/vehicle/animal) before sending to NVR — reduces NVR load and enables filtered event searching in UniFi Protect
- Packet Tracer simulation buffer setting: always enable "buffer filtered events only" + "auto clear event list" to prevent crashes
- DHCP (Dynamic Host Configuration Protocol) automatically assigns IP addresses when a device connects — your home router does this. Without a DHCP server you must statically configure IP addresses manually
- Every device on a network needs: IP address, subnet mask, default gateway (to reach other networks/internet), and DNS server (to resolve names to IPs)
- DNS (Domain Name System) converts human-readable names like google.com into IP addresses — devices don't actually use names to communicate, only IP addresses. nslookup lets you manually query your DNS server to resolve a name
- 169.254.x.x IP address means the device has no static IP and no DHCP server responded — called an APIPA address, signals a configuration problem
- `ping` sends an ICMP echo request and listens for an echo reply — fundamental tool for testing connectivity between two devices

## 🛠 Commands / Tools Practiced
- Cisco Packet Tracer — topology setup, adding devices, cabling, label management
- Packet Tracer preferences — font sizing for CLI readability, simulation buffer behavior settings
- Packet Tracer interface — logical vs physical view, device zoom, adding/removing modules (power supply, NIMs, WICs)
- `ipconfig` — shows IP address and subnet mask on a Windows device
- `ipconfig /all` — shows IP address, subnet mask, default gateway, DNS server, and MAC address
- `ping <IP address>` — tests communication between two devices (echo / echo reply)
- `nslookup <domain>` — resolves a domain name to an IP address using the configured DNS server (e.g. nslookup google.com returns Google's IP via 8.8.8.8)
