## 🧠 What I Learned / Key Takeaways
- Windows troubleshooting path for connectivity: loopback → gateway → 8.8.8.8 → google.com — work up the chain
- `net use` for mapped drives, `net user` and `net localgroup` for local account management — these are daily help desk commands
- DISM stands for Deployment Image Servicing and Management — not Dell-specific, works on all hardware
- Linux "everything is a file" is literal — /dev/sda is your disk, /proc/meminfo is live kernel RAM data, /dev/null discards output
- `/etc/hosts` is checked before DNS — if a hostname resolves wrong on one machine, check here first
- `ss -tuln` is the modern replacement for `netstat -ano` on Linux
- `tail -f` is one of the most important live troubleshooting commands on Linux
- On modern systemd Linux, `journalctl -u servicename` is more useful than grepping syslog
- SIGHUP (kill -1) reloads a service's config without restarting — prefer this over restart when only config changed
- SSH will refuse a key if permissions are too open — chmod 700 ~/.ssh and chmod 600 ~/.ssh/id_rsa
- AWS EC2 metadata is available at 169.254.169.254 — instance ID, AZ, IAM credentials, all accessible via curl from inside the instance
