### Key Concepts Drilled & Confirmed

**Cognito: User Pool vs Identity Pool**
- **User Pool** = Authentication only. Handles sign-up, sign-in, MFA, user directory. Does NOT issue AWS credentials.
- **Identity Pool** = Issues temporary AWS credentials. Supports authenticated users (social logins, SAML, OIDC) AND unauthenticated guest users.
- **Exam trigger:** "temporary AWS credentials" + "social media logins" + "guest users" = Identity Pool every time.
- Source: [AWS Cognito Docs](https://docs.aws.amazon.com/cognito/latest/developerguide/cognito-identity.html)

**Shared Responsibility Model: Host OS vs Guest OS on EC2**
- **Host OS** = AWS responsibility. AWS manages the host operating system and virtualization layer down to physical security.
- **Guest OS** = Customer responsibility. The customer manages the OS running inside their EC2 instance, including all patches and updates.
- Source: [AWS Shared Responsibility Model](https://aws.amazon.com/compliance/shared-responsibility-model/)

**Global Services (locked in)**
- IAM, CloudFront, Route 53, WAF (when attached to CloudFront)
- Everything else (EC2, RDS, DynamoDB, S3, etc.) is regional

**Serverless Services Master List (verified)**
- Lambda, DynamoDB (on-demand), S3, SQS, SNS, EventBridge, Fargate, Aurora Serverless, API Gateway, Cognito, Lambda@Edge
- NOT serverless: ElastiCache (requires node provisioning), EMR (requires cluster), OpenSearch (requires instances), RDS

**Load Balancer Types (verified)**
- **ALB (Application Load Balancer)** = Layer 7. Routes based on URL path, hostname, headers, cookies. Best for web apps and microservices.
- **NLB (Network Load Balancer)** = Layer 4. Routes based on IP + port. Ultra-low latency, supports static IP. Best for TCP/UDP and high-performance workloads.
- **GWLB (Gateway Load Balancer)** = Layer 3. Used for deploying third-party security appliances (firewalls, IDS/IPS). Transparent bump-in-the-wire inspection.
- All three fall under **Elastic Load Balancing (ELB)** as the umbrella service.

**Amazon Inspector (full definition)**
- Automated security assessment service that assesses applications for **exposure, vulnerabilities, and deviations from AWS best practices**.
- Covers vulnerability analysis AND compliance checking — not just CVEs.
- Exam trigger: "vulnerability analysis", "security audit", "assess for exposure and deviations from best practices"

**Security Services Distinction (confirmed)**
- **Inspector** = Proactive vulnerability and compliance assessment on EC2/containers
- **GuardDuty** = Continuous threat detection, monitors for malicious/unauthorized activity
- **Detective** = Post-incident investigation. Analyzes logs, finds root cause of security issues
- **Security Hub** = Central dashboard aggregating findings from GuardDuty, Inspector, Macie, and others
- **Macie** = Finds and protects sensitive data (PII) in S3

**Penetration Testing — Permitted Services Without Prior Approval**

- From this exam's answer choices: **RDS and S3** are correct
- Source: [AWS Penetration Testing Policy](https://aws.amazon.com/security/penetration-testing/)

**S3 Access Policy Types (verified)**
- **Bucket policies** = Resource-based policies attached directly to the S3 bucket. Can grant cross-account access.
- **IAM User policies (identity-based)** = Policies attached to IAM users, groups, or roles granting S3 permissions.
- Both are valid ways to grant access to S3 resources.
- "Object policies" is NOT a formal AWS policy type for S3.
- Source: [AWS S3 Policies and Permissions](https://docs.aws.amazon.com/AmazonS3/latest/userguide/access-policy-language-overview.html)

**Spot Instances**
- Price fluctuates in real time based on supply and demand of available EC2 capacity.
- Can be interrupted by AWS with 2-minute warning when capacity is needed.
- Exam trigger: "adjusts price based on supply and demand" = Spot Instance

**Convertible Reserved Instances vs Standard Reserved Instances**
- **Standard RI** = Highest discount (up to 72%), cannot change instance family/OS/tenancy
- **Convertible RI** = Slightly lower discount (up to 66%), CAN change instance family, OS, tenancy, and payment option. Exchange must result in equal or GREATER value — never lesser.

**DynamoDB — JSON Document Storage**

**IAM Group Characteristics (verified)**

- Groups CANNOT be nested (no group inside a group) — unlike Active Directory
- There is NO default group that includes all users

**AWS Backup**

**AWS Health**
- Notifies customers when AWS is experiencing events that may impact their AWS resources.
- Exam trigger: "AWS notifies customers about events impacting their resources" = AWS Health

- Requires MFA token to permanently delete S3 objects.

### Linux Networking — Concepts Covered

**NetworkManager vs Kernel vs udev**
- **Kernel** = Lowest level. Talks directly to hardware. Loads drivers, creates network interfaces. Names interfaces based on hardware MAC address.
- **udev** = Sits above kernel. Watches for hardware events and applies rules (e.g., rename interface when a specific MAC address is detected). Runs before anything else sees the device.
- **NetworkManager** = Top level (userspace). Manages connections — which network to join, credentials, auto-connect behavior. Does not touch hardware directly.
- **Flow:** Hardware → Kernel (loads driver, creates interface) → udev (applies rename rules) → NetworkManager (manages connection)

**Disabling Built-in WiFi via NetworkManager**
- Command used: `nmcli device set wlo1 managed no`
- Sets the built-in adapter to "unmanaged" — NetworkManager ignores it permanently, even after reboots
- USB adapter (connected as a different interface) becomes the only managed WiFi device

**udev Rules for Permanent Interface Rename**
- To permanently rename a network interface, create a udev rule in `/etc/udev/rules.d/`
- Rule syntax: `SUBSYSTEM=="net", ACTION=="add", ATTR{address}=="<MAC>", NAME="<new-name>"`
- After writing the rule: `sudo udevadm control --reload-rules` then reboot
