## Key Takeaways

### Billing & Pricing
- **AWS data transfer rule**: Data INTO AWS = free. Data OUT of AWS to 
  the internet = you pay. Data between services in the same region = 
  generally free. Data between regions = you pay.
- **Pay less by using more** applies to tiered consumption services like 
  S3, CloudFront, and data transfer — NOT raw EC2 scaling. Compute 
  discounts exist but are commitment-based (Savings Plans, RIs).
- **Support API** → Business or higher. Used to programmatically create 
  and manage support cases.
- **IEM (Infrastructure Event Management)** → Business (extra fee), 
  Enterprise On-Ramp (1 per year, included), Enterprise (unlimited, included).
- **Business-critical system down SLA** → does NOT exist at Business tier. 
  Starts at Enterprise On-Ramp (30 min) and Enterprise (15 min).
- **Production system down** (1 hr) and **Business-critical down** are two 
  separate severity levels. Production down = significant impact, 
  business still functioning. Business-critical = business is at risk, 
  existential urgency.
- **Response time SLAs** = promise to contact you, NOT a guarantee your 
  system will be fixed within that window.
- **Solutions Architect access** → Business or higher (separate from TAM).
- **TAM pool** → Enterprise On-Ramp. **Designated TAM** → Enterprise only.
- **Professional Services** → always a paid engagement, available to any 
  AWS customer regardless of support tier.

### CAF Perspectives — One-Liners
- **Business** → "is it worth it?" (ROI, KPIs, business value, outcomes)
- **People** → "do we have the right people and culture?" (roles, skills, 
  training, change management, workforce transformation)
- **Governance** → "are we doing this responsibly as an organization?" 
  (risk, compliance, guardrails, cloud program oversight, spending controls)
- **Platform** → "are we building it right?" (architecture, IaC, CI/CD, 
  hybrid infrastructure, multi-account design)
- **Security** → "are we protecting it?" (IAM, encryption, data protection, 
  security incident response)
- **Operations** → "are we running it right?" (monitoring, operational 
  incident response, event management, runbooks, continuous improvement)

### CAF Traps to Remember
- **Platform vs Operations**: Platform = designing and building. 
  Operations = monitoring and maintaining. "Architect" or "design" 
  in a question → Platform. "Monitor", "manage", "respond" → Operations.
- **Business vs Governance**: Business = value and ROI. Governance = 
  oversight and risk. "Is it worth it?" → Business. "Are we doing it 
  right?" → Governance.
- **Security vs Operations for incidents**: "Security incident response" 
  → Security. "Operational incident response" → Operations. The modifier 
  matters.
- **Governance is broader than compliance** — it covers the entire cloud 
  program oversight, not just regulatory requirements.

### Migration
- **Relocate vs Rehost**: Relocate = move the platform itself 
  (VMware on-prem → VMware Cloud on AWS). Rehost = lift and shift 
  workloads onto native AWS infrastructure (EC2). Both involve no code 
  changes. VMware in the question → Relocate.
- **SCT + DMS for heterogeneous migrations**: SCT converts the schema, 
  DMS moves the data. Homogeneous migrations → DMS only, no SCT needed.
- **Application Migration Service** → continuous replication, source stays 
  live during migration, minimal downtime cutover.

### Networking
- **Direct Connect** → dedicated physical private line from on-prem to AWS. 
  No internet involved. High cost, weeks to set up, consistent bandwidth.
- **Site-to-Site VPN** → encrypted tunnel over public internet, on-prem 
  to AWS. Fast to set up, cheaper, less reliable.
- **Client VPN** → individual users/devices connecting remotely to AWS 
  or on-prem resources. Think remote workers on laptops.
- **PrivateLink** → privately expose a service across VPCs or accounts 
  without touching the public internet. Underlying technology for 
  Interface Endpoints.
- **VPC Gateway Endpoint** → S3 and DynamoDB only, free.
- **VPC Interface Endpoint** → all other AWS services, powered by 
  PrivateLink, has a cost.

### Services
- **X-Ray** → distributed tracing. Use it when something is already 
  broken or slow. Shows request flow across services, identifies 
  bottlenecks. Think: "you only get an X-ray when something hurts."
- **Glue** → serverless ETL. "Serverless ETL" or "data catalog" in a 
  question → Glue.
- **EMR** → big data processing, Hadoop/Spark. "Hadoop" or "Spark" 
  in a question → EMR.
- **Kinesis** → real-time data streaming and ingestion, not ETL.
- **Amplify** → fully managed platform for deploying web/mobile apps. 
  Developers just write and ship code, AWS handles everything else.
- **WorkSpaces** → virtual desktops in the cloud, DaaS, remote workers.
- **IoT Core** → connect and manage IoT devices at scale. Device Shadow 
  = virtual representation of device state, works even when device is offline.
- **DynamoDB** → has two modes. On-demand = fully auto-scales, serverless. 
  Provisioned = you set capacity, can add Auto Scaling but not automatic 
  by default.
