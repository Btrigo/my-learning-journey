## 🧠 What I Learned / Key Takeaways
- Forward proxy = protects/represents the client; reverse proxy = protects/represents the server. Same mechanism, opposite side of the conversation being shielded.
- Global npm install just puts `claude` on PATH — it does NOT mean Claude Code has free rein over the whole file system. It only operates in the directory it's launched from, asks before file edits/bash commands by default, and permissions can be locked down further with an allow-list in `.claude/settings.json`.
- Claude Pro plan covers Claude Code in VS Code, Cursor, and JetBrains IDEs at no extra cost — IDE usage just draws from the same shared usage pool as regular Claude.ai chat, not a separate quota.
- Boris Cherny's top practical tips for Claude Code (translated into plain terms):
  - Start with codebase Q&A before ever editing anything — ask questions first, let Claude explore and explain (this is literally how Anthropic onboards new hires now; cut onboarding from 2-3 weeks to 2-3 days)
  - Ask Claude to brainstorm/make a plan before writing code on anything big, to avoid it building the wrong thing
  - CLAUDE.md = a cheat-sheet file automatically loaded every session — keep it short, put common commands/style guide/key files in it, check the shared one into git
  - Bash = the actual command language Claude runs for you (like IOS commands are to a router); safety is handled via read-only detection + tiered permission/allow-list system, not manual approval of every single command
  - MCP = a standard way to plug external tools (GitHub, databases, etc.) into Claude so it can use them directly
  - `claude -p` / the SDK = scripting mode, for automation pipelines — not a beginner concern yet
  - Parallel sessions / git worktrees = advanced/power-user pattern, running multiple Claude sessions on separate copies of the same repo — not something to worry about yet
  - Claude Code is fully multimodal — can take dragged/pasted images or file paths (e.g., hand it a UI mock to build against)
