## 🧠 What I Learned / Key Takeaways

### Elastic Load Balancer Types
- **ALB (Application Load Balancer)** = Layer 7, supports Lambda functions as targets. Trigger: "Lambda as target" or "application layer routing"
- **NLB (Network Load Balancer)** = Layer 4, ultra-low latency, TCP/UDP
- **Gateway Load Balancer** = third-party virtual appliances
- Only ALB supports Lambda as a target group

### CloudFormation
- Best tool to **replicate entire infrastructure** across regions — deploys everything (VPCs, EC2, RDS, IAM, security groups) as code
- Golden AMI only covers EC2 instances, not full infrastructure

### AWS Trusted Advisor
- **Basic/Developer:** 6 core checks only (S3 bucket permissions, security groups, MFA on root, EBS public snapshots, RDS public snapshots, service limits)
- **Business:** Minimum plan for **full** suite of checks across all 5 categories
- **Mnemonic:** "Can Pretty Seals Find Sardines" = Cost Optimization, Performance, Security, Fault Tolerance, Service Limits
- Concierge Support Team = **Enterprise On-Ramp and Enterprise** plans only

### Compute Savings Plans vs Reserved Instances
| | Compute Savings Plans | Reserved Instances |
|---|---|---|
| Covers Lambda? | ✅ Yes | ❌ No |
| Covers Fargate? | ✅ Yes | ❌ No |
| Covers EC2? | ✅ Yes | ✅ Yes |
| Flexibility | Any region, instance family, size | Locked to specific instance type/region |
| Max discount | Up to 66% | Up to 72% (Standard RI) |
| Commitment | 1 or 3 year spend commitment | 1 or 3 year instance commitment |

- **Exam trap:** EC2 + Lambda together → Compute Savings Plans only (RI doesn't cover Lambda)
- **Maximum EC2 discount:** Standard RI, 3-year, all upfront

### S3 Glacier Storage Classes — Retrieval Times
| Class | Retrieval Time | Best For |
|---|---|---|
| Glacier Instant Retrieval | Milliseconds | Archival data needing instant access |
| Glacier Flexible Retrieval | Minutes to 12 hours | Standard archival |
| Glacier Deep Archive | Up to 12 hours (standard) | Cheapest, long-term archival |

- **"12 hours or less + cheapest + long-term archival"** = Glacier Deep Archive
- Glacier Instant Retrieval is more expensive — only choose when millisecond retrieval is explicitly required

### AWS CloudTrail
- **Trail** (not "log") is the correct terminology
- All trails created via the **AWS Management Console are multi-region by default** — confirmed in AWS docs
- CLI/API creates single-region trails by default
- CloudTrail is **enabled by default** (90-day event history), but a trail is needed for longer retention
- First copy of management events delivered free in each region
- CloudTrail = **WHO did WHAT and WHEN** (API call auditing)
- Does NOT capture application error logs from EC2 (that's CloudWatch)

### CloudTrail vs CloudWatch
| | CloudTrail | CloudWatch |
|---|---|---|
| Purpose | API call auditing — who did what and when | Resource performance monitoring |
| Captures | AWS API calls, account activity | Metrics, logs, events from resources |
| Use case | "Who deleted that S3 bucket?" | "CPU is at 90%, trigger alarm" |
| Analogy | Security camera | Health monitor |
| Stores logs | S3 | CloudWatch Logs |

- **Exam trap:** "Application error logs from EC2" → CloudWatch, NOT CloudTrail

### Security Groups
- **Stateful** — return traffic automatically allowed
- **Default inbound:** All traffic DENIED
- **Default outbound:** All traffic ALLOWED
- Valid sources/destinations: IP addresses, CIDR ranges, security group IDs, prefix lists
- **Hostnames are NOT valid** sources or destinations
- Compare: NACLs are **stateless**, default allow all (custom NACL default = deny all)

### EFS (Elastic File System)
- **"Massive parallel access"** = EFS trigger phrase
- Managed NFS (Network File System) — allows thousands of concurrent connections
- Accessible from on-premises via Direct Connect or VPN
- Compare: EBS = attached to ONE EC2 instance; S3 = object storage; File Gateway = on-prem to S3 bridge

### Amazon EMR (Elastic MapReduce)
- **Trigger words:** Apache Spark, Apache Hadoop, Apache Hive, ETL, big data processing, data transformation, low-cost analytics framework
- Managed big data platform for running open-source frameworks
- Do NOT confuse with Redshift (data warehousing/SQL analytics)

### EBS Snapshots
- Point-in-time backups created via EBS snapshots
- Stored transparently in **Amazon S3** (you don't see the bucket)
- Subsequent snapshots are **incremental** — only changed blocks
- Instances do **NOT** need to be stopped to take a snapshot
- AMIs back up the entire instance, not just EBS volumes

### Amazon Cognito
- **User Pool** = authentication (sign-up, sign-in, social identity providers, SAML 2.0)
- **Identity Pool** = authorization (exchanges tokens for temporary AWS credentials)
- **Trigger:** "Mobile app + social identity providers + SAML 2.0" = Cognito User Pool

### IAM Policy
- **"Enforcing privileges and access controls"** = IAM Policy
- Policies are the documents defining what actions are allowed/denied on which resources
- Policies are attached to users, roles, and groups

### AWS Cost Explorer
- Provides **RI purchase recommendations based on past usage**
- Analyzes historical usage and recommends which RIs to buy
- Compare: Budgets = set alerts; Billing Dashboard = high-level overview; Cost and Usage Report = raw data dump

### AWS DataSync
- Managed service for automating and accelerating data transfers over the network
- Typically on-premises storage ↔ AWS (S3, EFS, FSx)
- Handles scheduling, encryption, verification, bandwidth throttling automatically
- **Trigger:** "Automate," "schedule," "on-premises to AWS," "online transfer"
- Compare: Storage Gateway = ongoing hybrid access; Snow Family = offline/physical transfer

### AWS CAF — People Perspective
- **"Organizational transformation," "team collaboration," "customer focus," "agile methodologies"** = People perspective
- Confirmed in AWS docs: organizing teams around products and value streams + agile methods = People perspective
- **Mnemonic:** "Big Purple Gorillas Play Soccer Often" = Business, People, Governance, Platform, Security, Operations

### AWS CAF — Governance Perspective Capabilities
- **Data Governance** ✅ = Governance perspective
- **Data Curation** ✅ = Governance perspective (selecting, organizing, maintaining data quality)
- **Data Protection** ❌ = Security perspective (not Governance)
- Full Governance capabilities: Program & Project Management, Benefits Management, Risk Management, Cloud Financial Management, Application Portfolio Management, Data Governance, Data Curation

### S3 Costs — What Incurs Charges
- **Storage** = charged per GB/month
- **Requests** = GET, PUT, LIST, COPY, etc. — charged per 1,000 requests
- **Egress (data OUT to internet)** = charged
- **Ingress (data IN from internet)** = FREE
- **Bucket policies, lifecycle policies, event notifications** = FREE (just configuration)
- **S3 Glacier Deep Archive retrieval** = charged per retrieval request

### 6 Advantages of Cloud Computing
1. Trade fixed expense for variable expense
2. Benefit from massive **economies of scale**
3. Stop guessing capacity
4. Increase speed and agility
5. Stop spending money on data centers
6. Go global in minutes

### ELB Health Checks
- First diagnostic for EC2 instances not reachable behind an ALB
- Health checks determine which instances receive traffic
- If failing health checks → ALB stops routing to that instance

### Amazon Pinpoint
- **"Digital user engagement"** + **"measure engagement across multiple channels"** (email, SMS, push notifications) = Pinpoint
- Marketing/engagement platform — NOT SNS (pub/sub), NOT SES (email only)

### Amazon Connect + Amazon Chime
- **Amazon Connect** = cloud contact center, customer service
- **Amazon Chime** = business communications (messaging, video, calls)
- Together: "improve business communication AND customer service experience"

### Throughput Optimized HDD (st1) EBS Volume
- Best for **throughput-intensive** workloads (streaming, big data, log processing)
- "Maximize performance AND cost" for high-throughput = st1
- Provisioned IOPS SSD = maximum IOPS but expensive; use only when IOPS (not throughput) is the requirement
