## 2. What I Learned / Key Takeaways

### Module 12 — Migration
- The **3 migration phases** are Assess → Mobilize → Migrate & Modernize, each with specific tools mapped to them
- **AWS CAF** has 6 perspectives: Business, People, Governance (business side) + Platform, Security, Operations (technical side)
- **7 Rs** of migration: Relocate, Rehost, Replatform, Refactor, Repurchase, Retain, Retire
- **DMS** moves data; **SCT** converts schema — heterogeneous migrations need both (SCT first)
- **DataSync** = automation/scheduling; **Transfer Family** = protocol support (SFTP/FTP/FTPS); **Direct Connect** = private dedicated connection
- **Snowball Edge** = physical device for offline petabyte-scale migrations

### Module 13 — Well-Architected Solutions
- **AWS Specialized Services** span 4 categories: Development (CodeBuild, CodePipeline, X-Ray, AppSync, Amplify), Business (Connect, SES), End-User Computing (AppStream 2.0, WorkSpaces, WorkSpaces Secure Browser), IoT (IoT Core)
- **IoT definition:** physical devices + sensors/software + communicates over internet = can be monitored/controlled remotely
- **Well-Architected Framework** has **6 pillars**: Operational Excellence, Security, Reliability, Performance Efficiency, Cost Optimization, Sustainability (**OSRPCS**)
- Each pillar maps to a real AWS service enhancement: Auto Scaling (OpEx), IAM/encryption (Security), CloudWatch (Reliability), Compute Optimizer (Performance), Savings Plans/Budgets/Cost Explorer (Cost), Cost & Usage Report (Sustainability)
- **WA Tool is free** — self-service, generates report with issues and remediations
- **Serverless web backend pattern:** API Gateway → Lambda → DynamoDB, monitored by X-Ray
- **Serverless static website:** S3 + API Gateway + Lambda + SES
- **Smart customer support (callback option):** Amazon Connect + Lambda + CloudFront — customers on hold can transition from voice to chat/email or request a callback instead of waiting
