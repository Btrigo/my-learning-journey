## üß† What I Learned / Key Takeaways
- `.bashrc` is a configuration file that runs every time you open Git Bash. By adding an alias to .bashrc, you create a shortcut command that maps a simple word (like newlog) to a full command or script path. When you type that alias from any directory, the computer looks up what the alias points to and executes it. This is different from adding something to PATH. an alias is a direct shortcut to one specific command, while PATH tells the computer which folders to search through when looking for any command.
- `.bashrc` is a configuration file that runs every time you open Git Bash - it's where you store shortcuts (aliases) and settings
- `source ~/.bashrc` reloads the settings without closing the terminal
- Bash scripts need `chmod +x` to become executable
- `${1:-default}` syntax in bash means "use first argument, or use default if nothing provided"
- `$(date +%Y-%m-%d)` generates today's date in YYYY-MM-DD format
- Aliases are shortcuts that let you type simple commands instead of long paths
