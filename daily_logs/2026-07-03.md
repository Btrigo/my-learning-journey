## 🧠 What I Learned / Key Takeaways
- **Cloud logging limitations**: unlike on-prem, you can't just install a SIEM agent in the cloud — you're dependent on the cloud provider's native logging (e.g., AWS CloudTrail). Common pain points: paid logging tiers (SIEM integration may require extra license/cost), poor/unstructured log formats, and some SaaS platforms simply not supporting SIEM integration at all
- **Moving an unpatched server to the cloud does NOT make it secure** — the vulnerability moves with it; the cloud provider secures the underlying infrastructure, not your unpatched software (this is the shared responsibility model in practice)
- Reviewed real-world incident examples: Capital One breach (AWS S3 misconfiguration exposed ~1M SSNs and 140k SSNs from credit card customers) and general SaaS breach patterns (weak passwords, leaked API keys, stolen cookies as common root causes)
- Started "Cloud Computing Fundamentals" — framed around the practical problem cloud solves: hosting an app on your own machine breaks down under global latency, concurrent users, and single points of failure (your computer being off = app down)
- Terraform IaC structuring reinforcement: `variables.tf` holds reusable values referenced across multiple modules (avoids repeating hardcoded values); `main.tf` acts as the central file where all defined modules get referenced together — variables get called into resource blocks using `var.<name>` syntax
