## 🧠 What I Learned / Key Takeaways
- chmod changes what's allowed (permission bits), chown changes who owns it (user/group) — diagnose which one is actually broken before touching either
- usermod -aG appends a group; leaving off -a replaces the entire supplementary group list, wiping out existing memberships
- Group/permission changes (Linux usermod, Windows AD group membership) don't apply to an already-active session — needs a new login or newgrp
- /etc/passwd only shows primary group; groups/id resolve full supplementary group membership properly
- Docker's overlay2 storage accumulates old image layers and stopped containers by default — docker system df before docker system prune -a --volumes, since the latter is genuinely destructive
- ver and set are cmd.exe-only built-ins, not standalone executables — fail in PowerShell, which has its own equivalents
