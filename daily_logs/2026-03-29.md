## 🧠 What I Learned / Key Takeaways
- AWS Billing Console / Cost Explorer is for viewing and analyzing RI usage; 
  Organizations handles RI sharing across accounts but not detailed reporting
- AWS Glue is the managed ETL service — extracts from S3, RDS, Redshift, DynamoDB,
  transforms, and loads into a data warehouse or data lake; Redshift can be both source
  and destination
- Data lake (S3) = raw unstructured storage; Data warehouse (Redshift) = clean,
  structured, analytics-ready; RDS = OLTP, Redshift = OLAP
- Hybrid architecture: move compute to AWS (EC2), keep data on-prem, connect via
  Direct Connect or Site-to-Site VPN; Storage Gateway extends on-prem storage to
  cloud without migrating data
- CloudFront handles both static and dynamic content delivery globally — Route 53
  routes traffic, CloudFront delivers content
- AZs are within ~60 miles for low-latency backend replication, not end-user latency;
  end-user latency is handled by CloudFront edge locations
- AWS SDKs provide language-specific libraries for programmatic access to AWS at
  runtime — every CLI command, console click, and SDK call is ultimately an HTTPS
  API call signed with credentials
- CodePipeline is AWS's CI/CD orchestration service — connects CodeCommit, CodeBuild,
  and CodeDeploy into automated release pipelines
- S3 Standard-IA is correct for infrequently accessed but immediately retrievable
  data; Glacier is archival only — retrieval delay disqualifies it for active workloads
- Oracle is the only RDS engine supporting BYOL — proprietary licensing;
  Aurora supports MySQL and PostgreSQL compatibility only, is self-healing with
  6 copies across 3 AZs, survives loss of 2 copies for writes and 3 for reads
- RDS Read Replicas increase read throughput; Multi-AZ increases availability/failover
- Enterprise support SLA: 15 minutes for business-critical, 1 hour for production down
- CAF perspectives: Business, People, Governance, Platform, Security, Operations
  (Big Purple Gorillas Play Soccer Often)
- Well-Architected pillars signal words — Cost Optimization: right-sizing, right number
  of resources, eliminating waste; Operational Excellence: runbooks, automation,
  improving processes; Reliability: recover from failure; Performance Efficiency:
  best resource for performance outcomes; Sustainability: environmental impact
- Cost management tools: CUR = granular data + exports to Athena/Redshift/QuickSight;
  Cost Explorer = visualize trends; Budgets = alerts on thresholds;
  Cost Management Dashboard = high-level overview
- Global services: IAM, Route 53, CloudFront, WAF (when attached to CloudFront)
- AWS Outposts = same AWS APIs on-prem for hybrid/compliance requirements;
  Local Zones = low latency near cities; Wavelength = 5G mobile edge;
  Dedicated Hosts = software licensing compliance on dedicated hardware in AWS
- Economies of scale = AWS passes hardware cost reductions to customers annually
- Bastion host = jump server in public subnet for SSH access to private instances;
  SSM Session Manager eliminates the need for bastion hosts entirely
