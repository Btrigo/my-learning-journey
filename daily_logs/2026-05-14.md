## 🧠 What I Learned / Key Takeaways
- **Career clarity:** The play is build AWS projects + work toward SAA → land entry-level IT role by Sept/Oct → transition into cloud from employment. CCNA shelved for now but not abandoned.
- **SAA reality check:** The SAA cert alone won't land a Solutions Architect role — that's a senior title. The obtainable roles are Cloud Engineer and Cloud Developer. Certs are a structured learning goal, not a job title.
- **OAC (Origin Access Control):** CloudFront signs every request to S3 using SigV4. The S3 bucket policy enforces that only requests from the specific CloudFront distribution ARN are honored. Direct S3 access returns AccessDenied. OAC replaced legacy OAI and is current AWS best practice.
- **Linux troubleshooting fundamentals:** `dmesg` is the first stop for hardware issues. DKMS "added" does not mean "installed" — added means registered, installed means compiled and ready. Kernel updates can silently break out-of-tree drivers.
- **modprobe:** Manually loads kernel modules and resolves the full dependency chain. Useful for restoring hardware without a reboot but doesn't persist unless properly configured.

## 🛠 Commands / Tools Practiced
- `dmesg | grep [driver]` — kernel message filtering for hardware debugging
- `sudo modprobe [module]` — manual kernel module loading
- `sudo update-initramfs -u -k all` — apply kernel config changes to all installed kernels
- `printf "module\n" | sudo tee /etc/modules-load.d/file.conf` — persist module loading across reboots
