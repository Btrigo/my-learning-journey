## 🧠 What I Learned / Key Takeaways
- GuardDuty ingests VPC Flow Logs, CloudTrail, and DNS automatically — 
  no agent or infrastructure setup required, but optional configuration 
  (custom threat lists, suppression rules, EventBridge integration) is available
- GuardDuty detects but does not block — it is not a firewall
- Gateway endpoints = S3 and DynamoDB only, free, work via route table modification
- Interface endpoints = most other AWS services, have a cost, use an ENI 
  with a private IP, and work across VPCs and on-prem via VPN/Direct Connect
