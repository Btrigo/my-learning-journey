## üß† What I Learned / Key Takeaways
- **Shebang importance**: The `#!/bin/bash` line must be exact - missing the leading `/` causes "bad interpreter" errors
- **Script execution methods**: `./script.sh` relies on shebang and execute permissions, while `bash script.sh` explicitly calls the interpreter
- **Why `./` is needed**: The shell only searches PATH directories by default; `./` tells it to look in the current directory
- **Shell comparison**: Bash is widely compatible for scripting, Fish is beginner-friendly with auto-correction, Zsh offers advanced customization
- **Core scripting concepts**: Variables store reusable values, loops automate repetitive tasks, conditionals control program flow based on conditions
- **Best practice**: Always use `chmod +x` to make scripts executable properly
