## 🧠 What I Learned / Key Takeaways
- AWS Regions = geographic separation; AZs = separate data centers within a region with low latency
- VPN = encrypted tunnel over internet; Direct Connect = private but not encrypted by default
- Session Manager = EC2 access without SSH keys or open inbound ports
- CloudTrail = API activity logging; CloudWatch = performance metrics; Config = configuration change tracking; Trusted Advisor = best practice checks including service limits
- IAM roles are used when AWS services need to access other AWS services — not users, not groups
- CLI authentication uses IAM access keys, not SSH keys or passwords
- Root user access keys should be deleted; root is only for account-level actions
- Elastic Beanstalk handles environment creation and app deployment automatically
- Lambda billing = number of requests + duration
