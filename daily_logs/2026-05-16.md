## 🧠 What I Learned / Key Takeaways

- **S3 versioning cannot be fully disabled** — only suspended; existing versions persist; Delete Markers are not permanent deletes
- **MFA Delete** requires the root account to enable; protects against accidental or malicious permanent deletion of versioned objects
- **Replication requires versioning on both source AND destination buckets**; existing objects need S3 Batch Replication; Glacier Flexible/Deep Archive objects cannot be replicated
- **S3 Intelligent-Tiering** does not move objects smaller than 128KB — they stay in Frequent Access tier permanently regardless of access patterns
- **Block Public Access** overrides any permissive bucket policy — account/bucket level kill switch; enabled by default on new buckets
- **Object Lock Compliance mode** means nobody — including root — can delete before retention expires; Governance mode allows override with a special IAM permission
- **NAT Gateway** scales 5–100 Gbps automatically but is only resilient within a single AZ — need one per AZ for full fault tolerance
- **NAT Gateway cannot route traffic over VPC Endpoints, VPN, Direct Connect, or VPC Peering** — those routes go directly in the route table
- **Security Groups are stateful** (return traffic auto-allowed); **NACLs are stateless** (must explicitly allow both directions); Security Groups can only allow — only NACLs can explicitly deny
- **Gateway Endpoints** (S3 and DynamoDB) are free and added to the route table; **Interface Endpoints** (everything else) create an ENI and cost money — exam almost always wants Gateway for S3/DynamoDB
- **VPC Peering is non-transitive** — A↔B and B↔C does not mean A can reach C; Transit Gateway solves this with hub-and-spoke transitive routing
- AWS reserves **5 IPs per subnet** — always subtract 5 from the total when calculating usable addresses
