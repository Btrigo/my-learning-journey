**New or reinforced concepts:**
- **Snow Family** — Snowcone (space-constrained/field, 8–14TB), Snowball Edge (large migrations, 80–210TB), Snowmobile (exabytes); DataSync built into Snowcone for when connectivity is available
- **DataSync vs Storage Gateway vs AWS Backup vs Elastic Disaster Recovery** — DataSync = online transfer; Storage Gateway = hybrid connectivity; AWS Backup = centralized backup policy management; EDR = full app recovery after disaster
- **Cognito User Pool vs Identity Pool** — User Pool = authentication/sign-in; Identity Pool = temporary AWS credentials + guest/federated access
- **EC2 pricing** — Spot pricing adjusts based on supply and demand, up to 90% off, 2-min interrupt warning; Convertible RI exchange must result in equal or greater value; Standard RI = biggest discount, no flexibility
- **ELB types** — ALB (Layer 7, HTTP/HTTPS), NLB (Layer 4, TCP/UDP/TLS, ultra-low latency, millions req/sec), GLB (Layer 3, virtual appliances/firewalls)
- **Serverless ≠ Managed** — RDS and standard Aurora are managed but NOT serverless; serverless services include Lambda, Fargate, S3, DynamoDB (on-demand), SQS, SNS, API Gateway, EventBridge, Step Functions, Aurora Serverless, Cognito, Kinesis
- **SOA = decouple** — Service-Oriented Architecture signal word always maps to decoupling components
- **Security services** — GuardDuty = finds threats; Detective = investigates root cause; Security Hub = aggregates findings; Inspector = vulnerability scanning; Macie = PII/sensitive data discovery in S3
- **IAM Policy Simulator** — tests and troubleshoots IAM and resource-based policies before applying them
- **CAF Security capabilities** — Infrastructure Protection (external threats/unauthorized access), Data Protection (data at rest/transit), IAM, Threat Detection, Application Security, Incident Response
- **AWS Health** — personalized notifications when AWS events impact YOUR specific resources
- **MFA Delete** — requires MFA device to permanently delete S3 objects; credentials alone are not enough
- **Elastic IP** — NOT required to launch EC2; only needed for a static persistent IP
- **Service Quotas** — exist for all accounts regardless of plan; most are soft limits raiseable via AWS Support; Trusted Advisor shows proximity to limits but cannot raise them
- **Penetration testing** — permitted without prior approval on: EC2/NAT/ELB, RDS, CloudFront, Aurora, API Gateway, Lambda/Lambda@Edge, Lightsail, Elastic Beanstalk; S3 is NOT permitted; DoS/DDoS simulation always prohibited
- **Enterprise support exclusive features** — Well-Architected Reviews, Operations Reviews, dedicated TAM; IEM available at Business (extra fee) or Enterprise (included)
- **TCO (Total Cost of Ownership)** — moving to AWS decreases TCO by eliminating hardware, power, cooling, maintenance, and staffing costs
- **Cross-AZ data transfer** — $0.01/GB each direction; same-AZ = free; inbound to AWS = always free; CloudFormation service itself = free
- **S3 Multipart Upload** — recommended for objects 100MB+; parallel uploads improve throughput and resilience to network failures
- **Aurora engines** — MySQL and PostgreSQL only; never SQL Server or Oracle (that's standard RDS)
