## 🧠 What I Learned / Key Takeaways
- Am I following all WAF best practices? → AWS Well-Architected Tool
- Are my instances the right size? → AWS Compute Optimizer
- Am I spending efficiently? → AWS Cost Explorer
- Broad cost and usage recommendations? → AWS Trusted Advisor (full checks at Business tier and above)
- Trusted Advisor 7 core checks (Basic/Developer): 6 security checks (S3 permissions, Security Groups, IAM use, MFA on root, EBS public snapshots, RDS public snapshots) + 1 service limits check
- Support plan critical SLAs: Business = 1hr, Enterprise On-Ramp = 30min, Enterprise = 15min for business-critical
- TAM: pool at Enterprise On-Ramp, dedicated at Enterprise only
- IEM (Infrastructure Event Management): extra fee at Business, included at Enterprise tiers — proactive support for planned high-traffic events
- SES = application-level email sending (transactional + bulk); Lambda structures the email, SES handles delivery
- Amazon Connect = cloud contact center service (IVR is one component, not the whole thing)
- AWS Amplify = tools for front-end web and mobile developers to build scalable full-stack apps; auto-provisions Cognito, AppSync, S3, CI/CD under the hood
- Elastic Beanstalk = PaaS for deploying and scaling web applications; handles EC2, load balancing, auto scaling — you bring the app code
- Amplify vs. Beanstalk: Amplify targets front-end/mobile devs building new apps; Beanstalk targets developers with existing server application code
- Athena = serverless interactive query service; runs SQL directly against data in S3; pay per query ($5/TB scanned); supports structured, semi-structured, and unstructured formats
- Redshift architecture: Leader Node coordinates, Compute Nodes execute queries in parallel (MPP), Redshift Spectrum queries S3 directly without loading data
- CodePipeline = continuous delivery service; orchestrates the full release pipeline (source → build → test → deploy); CodeBuild is one step inside it
- CAF Platform perspective = build it (architect, provision, IaC, CI/CD); CAF Operations perspective = run it (monitor, incident response, recovery)
- Rehost vs. Relocate: Rehost moves anything (physical, virtual, cloud) as-is; Relocate moves a platform to its cloud equivalent (e.g., VMware SDDC → VMware Cloud on AWS)
- Repurchase vs. Retire: Repurchase = still need the function, switching to a different product (SaaS); Retire = no longer needed, shut it down
