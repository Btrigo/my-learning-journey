## 🧠 What I Learned / Key Takeaways
- EBS snapshots are incremental and stored in S3 transparently
- Security group rules require IP/CIDR or security group ID as source — never hostname or instance ID
- Compute Savings Plans cover EC2 + Lambda + Fargate — Reserved Instances cover EC2 only
- CAF Align phase = gap analysis and cross-org dependencies (not Envision)
- Throughput Optimized HDD (st1) is for streaming/sequential workloads — not Provisioned IOPS SSD
- CloudTrail trail created in console applies to all regions by default — first trail is free
- Step Functions = coordinate multiple AWS services into serverless workflows
- Firewall Manager = centrally enforce rules across accounts; Security Hub = centrally view findings
- S3 Block Public Access overrides all bucket policies and ACLs
- Performance Efficiency vs Cost Optimization: intent determines the pillar — "right resource" vs "reduce spend"
- IaC = Operational Excellence, not Performance Efficiency
- Reserved Instances only cover infrastructure-level services (EC2, RDS, ElastiCache, Redshift, DynamoDB) — not abstraction layers like Beanstalk, EKS, or Batch
