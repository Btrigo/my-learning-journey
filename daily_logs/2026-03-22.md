## 🧠 What I Learned / Key Takeaways
- **Linux diagnostics:**
    - `lspci` identifies PCIe hardware, `dmesg` reads kernel logs, `rfkill` manages wireless kill switches
    - Intel 9462 CNVi has a known TLV/NVM LAR conflict under Linux — deep firmware/driver compatibility issue
    - `/etc/modprobe.d/` stores persistent kernel module options — changes survive reboots
    - systemd services can be created to run commands automatically at boot
- **Web servers:**
    - Apache uses process-per-connection model — struggles under high concurrency
    - Nginx uses event-driven, non-blocking architecture — one worker handles thousands of connections
    - Virtual hosts = multiple websites on one server, routed by the HTTP Host header
    - Reverse proxy = Nginx sitting in front of an app, handling SSL, load balancing, caching
- **Event-driven / non-blocking:**
    - Single-threaded (Node.js) doesn't mean slow — it means never idle, always reacting to events
    - Callbacks fire when events happen rather than blocking and waiting
    - JavaScript in the browser uses the same model — click handlers, API calls, all event-driven
- **AWS CloudFront** = AWS's CDN — caches content at edge locations globally for low latency
- **Control plane vs data plane:**
    - Control plane = the brain — makes decisions, builds rules, manages configuration
    - Data plane = the muscle — actually forwards/processes traffic based on control plane rules
    - In AWS: console/API = control plane, EC2/S3 doing actual work = data plane
- **Linux filesystem hierarchy:**
    - `/etc` = config files, `/var` = logs/data, `/bin` = programs, `/lib` = libraries, `/home` = user files

## 🛠 Commands / Tools Practiced
- `lspci | grep -i network` — identify network hardware
- `lsmod | grep iwlwifi` — check if WiFi driver is loaded
- `sudo dmesg | grep -i iwlwifi` — read kernel log for driver errors
- `rfkill list` — check wireless kill switch status
- `sudo rfkill unblock wifi` — lift software WiFi block
- `ip link show wlo1` — check interface state
- `sudo ip link set wlo1 up` — bring interface up
- `sudo modprobe -r iwlwifi && sudo modprobe iwlwifi` — reload WiFi driver
- `nmcli device status` — check NetworkManager device states
- `nmcli device wifi list` — scan for available WiFi networks
- `sudo nano /etc/modprobe.d/iwlwifi.conf` — edit kernel module options
- `sudo nano /etc/systemd/system/rfkill-unblock.service` — create systemd service
- `sudo systemctl enable/start/status <service>` — manage systemd services
