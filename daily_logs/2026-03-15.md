## 🧠 What I Learned / Key Takeaways
- **Region selection priorities:** Compliance requirements take precedence, followed by proximity to users, feature availability, and pricing considerations.
- **CloudFormation** enables Infrastructure as Code (IaC), allowing infrastructure to be defined in templates and deployed consistently across environments.
- **Public vs Private Subnets:** A subnet is considered public when its route table includes a route to an Internet Gateway; private subnets have no direct route to the internet.
- **NAT Gateway vs Virtual Private Gateway:** NAT Gateways allow private instances to initiate outbound internet connections, while Virtual Private Gateways connect VPCs to on-premises networks via VPN.
- **Direct Connect vs VPN:** Direct Connect provides a dedicated private fiber connection to AWS for hybrid environments requiring consistent bandwidth and lower latency.
- **Security Groups vs Network ACLs:** Security groups are stateful instance-level firewalls, while NACLs are stateless subnet-level controls that evaluate every packet in both directions.
- **Edge services distinctions:** Route 53 handles DNS routing, CloudFront distributes cached content through a CDN, and Global Accelerator routes application traffic over the AWS global backbone for improved availability and performance.
