## 🧠 What I Learned / Key Takeaways
- Root guard is placed on ports facing untrusted neighbors — NOT on uplinks between legitimate switches. Enables on those ports would block your own topology
- Root guard puts a port into root inconsistent state (auto recovery) — different from BPDU guard which error-disables (manual recovery)
- Setting your root bridge priority to 0 is not a security measure — an attacker can spoof a lower MAC address and still win the STP election. Root guard is the actual defense
- STP root bridge hijacking is a real attack vector — attacker sets priority 0, wins election, all traffic flows through their switch, SPAN/port mirroring gives full visibility into traffic
- The extended system ID is why show spanning-tree shows 4097 and 8193 instead of 4096 and 8192 — VLAN number is added to the base priority
- BPDU filtering globally vs per interface is a critical distinction — global is safer and self-corrects, per interface effectively disables STP on that port and can destroy the network
- Loop guard protects against unidirectional fiber link failures — if BPDUs stop being received, port transitions to loop inconsistent state instead of forwarding
- Unidirectional link failures are a physical problem that can happen any time on fiber — BPDU filtering in the demo was just simulating it in a lab
- Loop guard and root guard are mutually exclusive on the same port — loop guard wins if both are configured
- Alternate ports are elected automatically by STP based on path cost and Bridge ID — not manually configured
- OSI model data units: data (L7-5), segment (L4), packet (L3), frame (L2), bits (L1)
- Everything covered in STP lives at Layer 2 — MAC addresses, frames, VLANs, trunking
- Native VLAN is a Cisco term for the 802.1Q untagged VLAN on a trunk — other vendors call it untagged VLAN but the behavior is identical
- Cisco networking concepts are vendor-neutral at the concept level — commands differ but the underlying IEEE standards are universal

## 🛠 Commands / Tools Practiced
- `spanning-tree vlan 1 priority 0` — set STP priority (attacker scenario)
- `spanning-tree vlan 1 root primary` / `root secondary` — shortcut to set root and backup root
- `interface range gigabitethernet 1/0/1 - 24` + `spanning-tree guard root` — enable root guard across port range
- `spanning-tree loopguard default` — enable loop guard globally on all ports
- `spanning-tree guard loop` — enable loop guard on a specific interface
- `spanning-tree bpdufilter enable` — enable BPDU filtering per interface (dangerous in production)
- `spanning-tree portfast bpdufilter default` — enable BPDU filtering globally (safer, self-correcting)
- `spanning-tree portfast` — enable PortFast on an interface
- `spanning-tree portfast default` — enable PortFast globally on all access ports
- `spanning-tree portfast bpduguard default` — enable BPDU guard globally on all PortFast ports
- `show spanning-tree` / `show spanning-tree vlan 1` — verify STP topology, port roles, port states
- `show processes cpu history` — check switch CPU utilization (demonstrated during broadcast storm)
- `show interface gigabitethernet 1/0/1` — verify traffic rates (demonstrated at 700Mbps during loop)
