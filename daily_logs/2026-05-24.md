## 🧠 What I Learned / Key Takeaways
- **File separation**: splitting by concern across 5 files is standard professional practice — files are for engineers, not for Terraform
- **`providers.tf` first**: provider is the foundation everything else depends on — define it before any resources
- **HCL block syntax**: every block follows `block_type "label" { }` — curly braces define scope, nesting reflects hierarchy
- **Resource references**: `resource_type.local_name.attribute` — Terraform resolves dependency order automatically
- **`jsonencode()`**: S3 bucket policies must be JSON — `jsonencode()` converts HCL to JSON automatically
- **IAM policy structure**: Version, Statement, Effect, Principal, Action, Resource, Condition — keys are capitalized, AWS-defined values must match exactly
- **`StringEquals` condition**: locks bucket access to one specific CloudFront distribution — not any CloudFront, only yours
- **CloudFront Function URI rewriting**: rewrites directory-style requests to `index.html` before hitting S3 — assets with a `.` are passed through untouched
- **`force_destroy = true`**: has zero effect while infrastructure is running — only matters at destroy time, empties and deletes bucket in one shot
- **`.terraform.lock.hcl` should be committed**: locks provider version for reproducibility across environments
- **`terraform fmt`**: auto-formats all `.tf` files — files not listed in output were already clean
- **`terraform validate`**: requires `terraform init` first — provider must be downloaded before syntax can be checked
- **Type keywords vs string values**: `type = string` has no quotes — `string` is a keyword, not a value
- **Line-by-line code breakdowns**: the most effective learning method — "you're literally saying..." explanations make syntax stick
